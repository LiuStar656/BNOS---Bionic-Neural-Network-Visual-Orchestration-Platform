# BNOS 菜单栏重构方案

## 📋 设计目标

将工具栏的所有功能整合到菜单栏中，实现更简洁的界面布局和更符合传统桌面应用习惯的操作方式。

## 🎯 菜单结构设计

### 1. 文件菜单 (File) - `Alt+F`

**职责**: 项目管理 + 视图控制 + 应用设置

| 菜单项 | 快捷键 | 功能说明 | 原位置 |
|--------|--------|----------|--------|
| 📁 新建项目 | Ctrl+N | 创建新的神经网络项目 | 工具栏 |
| 📂 打开项目 | Ctrl+O | 打开现有的神经网络项目 | 工具栏 |
| --- | --- | 分隔线 | --- |
| 📋 节点列表 | - | 显示/隐藏节点列表面板（可勾选） | 工具栏 |
| --- | --- | 分隔线 | --- |
| 🎨 颜色设置 | - | 自定义画布和节点颜色 | 工具栏 |
| --- | --- | 分隔线 | --- |
| ❌ 退出 | Ctrl+Q | 退出应用程序 | 原有 |

**设计理念**:
- 将所有"项目级别"的操作放在文件菜单
- 视图控制（节点列表开关）属于项目视图管理，放在文件菜单
- 颜色设置属于应用级配置，也放在文件菜单

---

### 2. 编辑菜单 (Edit) - `Alt+E`

**职责**: 节点操作 + 画布编辑

| 菜单项 | 快捷键 | 功能说明 | 原位置 |
|--------|--------|----------|--------|
| ➕ 新建节点 → | - | 子菜单：选择语言创建节点 | 工具栏 |
| ├─ Python | - | 创建Python节点 | 工具栏下拉框 |
| ├─ Node.js | - | 创建Node.js节点 | 工具栏下拉框 |
| ├─ Go | - | 创建Go节点 | 工具栏下拉框 |
| ├─ Java | - | 创建Java节点 | 工具栏下拉框 |
| ├─ C++ | - | 创建C++节点 | 工具栏下拉框 |
| ├─ Rust | - | 创建Rust节点 | 工具栏下拉框 |
| └─ Shell | - | 创建Shell节点 | 工具栏下拉框 |
| --- | --- | 分隔线 | --- |
| 🔄 刷新节点 | F5 | 刷新节点列表 | 工具栏 |
| 🗑️ 清空连线 | - | 清空所有节点连线 | 工具栏 |
| --- | --- | 分隔线 | --- |
| ▶️ 启动节点 | Ctrl+Shift+S | 启动选中的节点 | 工具栏 |
| ⏹️ 停止节点 | Ctrl+Shift+X | 停止选中的节点 | 工具栏 |

**设计理念**:
- "新建节点"作为子菜单，替代原来的语言下拉框+按钮组合
- 所有节点生命周期管理（创建、刷新、启停）集中在编辑菜单
- 符合"编辑"语义：对节点进行各种操作

## 📊 对比分析

### 原方案（工具栏 + 菜单栏）
**优点**:
- 常用功能一键可达
- 可视化程度高

**缺点**:
- 占用垂直空间
- 功能重复（工具栏和菜单都有）
- 语言选择需要两步操作（下拉框+按钮）
- 界面元素过多，视觉杂乱

### 新方案（纯菜单栏）
**优点**:
- ✅ 界面更简洁清爽
- ✅ 符合传统桌面应用习惯
- ✅ 功能组织更有逻辑性
- ✅ 快捷键提升效率
- ✅ Emoji图标弥补视觉提示
- ✅ 节省垂直空间，画布更大

**缺点**:
- ⚠️ 需要多一次点击访问功能
- ⚠️ 新用户可能需要学习菜单位置

**权衡**: 
- BNOS是专业工具，用户会频繁使用，熟悉菜单后效率更高
- 快捷键可以弥补点击次数增加的问题
- 画布空间比快速访问更重要

## 📌 总结

本次重构将BNOS从"工具栏+菜单栏"的双层交互模式简化为纯菜单栏模式，主要收益：

1. **界面更简洁**: 释放约80-100px垂直空间
2. **逻辑更清晰**: 功能按语义分组，符合Mental Model
3. **操作更高效**: 快捷键+子菜单，减少鼠标移动距离
4. **维护更容易**: 单一入口点，代码更集中
