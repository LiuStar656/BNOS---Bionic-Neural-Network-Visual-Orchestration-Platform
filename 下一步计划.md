## 二、设计准则

1. **容器与逻辑分离**：面板/画布/窗口只做视图挂载，不承载业务代码
2. **功能原子化**：单一职责，一个组件只做一件事
3. **业务下沉管理层**：数据/进程/项目逻辑统一归入 Manager
4. **消灭重复**：通用工具函数全局唯一入口
5. **新增功能只增文件、不动旧代码**

## 五、每次提取的标准流程

1. 创建新模块文件，复制目标方法/类
2. 添加必要的 import
3. 在原文件中导入新模块，替换调用为委托
4. 运行 `python -c "import py_compile; py_compile.compile(...)"` 验证
5. 删除原文件中的旧代码
6. 启动 GUI 冒烟测试

---

## 六、注意事项

1. **不改逻辑**：只做文件拆分和 import 调整，不动函数签名和业务逻辑
2. **逐步提交**：每完成一个文件拆分后 git commit，方便回滚
3. **先消除重复后拆分**：file_utils/log_viewer 优先做，避免拆出重复代码
4. **向后兼容**：旧 import 路径通过 `__init__.py` 重导出保持可用
5. **i18n 不受影响**：所有 `t()` 调用无需修改，`strings_cn.json` 不动
