## 1. 画布节点展开功能（新组件）

### 需求描述

在画布节点（`NodeItem`）上添加一个展开按钮，点击后弹出节点详情窗口。窗口左侧为 `output.json` 可编辑文本框，右侧为启动/停止、节点配置、删除节点三个按钮。

### 设计原则

- **非侵入式**：以新组件方式实现，不改动现有 `NodeItem` 核心逻辑
- **组件独立**：新建 `ui/panels/node_expand_panel.py`

### NodeItem 改动（最小化）

在 `NodeItem` 的 `mouseDoubleClickEvent` 或右键菜单中新增触发入口。不改构造函数，仅新增一个回调信号：

```python
# ui/canvas/items/node_item.py 末尾追加
expand_requested = pyqtSignal(str)  # 节点名称
```

在 `NodeItem.mousePressEvent` 中判断是否点击了展开图标区域（矩形右上角 16×16），若是则 emit `expand_requested`。

### 新组件：NodeExpandPanel

**文件**：`ui/panels/node_expand_panel.py`

**类名**：`NodeExpandPanel(QDialog)`

**窗口类型**：与 `NodeConfigDialog` 同（标准 `QDialog`，模态）

**布局**：
```
┌─────────────────────────────────────────┐
│ 节点详情 - {node_name}                   │ ← 窗口标题
├──────────────────┬──────────────────────┤
│                  │  启动节点              │
│                  │  停止节点              │  ← 右侧按钮组
│  output.json     │  ──────────           │
│  (QTextEdit)     │  节点配置              │
│  可编辑          │  删除节点              │
│                  │                       │
│                  │                       │
├──────────────────┴──────────────────────┤
│  保存              关闭                  │ ← 底部按钮
└─────────────────────────────────────────┘
```

**实现要点**：
- 左侧 `QTextEdit`（等宽字体、深色背景），默认只读，旁边放一个小编辑/锁定切换按钮
- `output.json` 不存在时显示"暂无输出数据"
- 右侧三个按钮垂直排列，间隔 10px
- 启动/停止按钮根据节点运行状态动态切换文本
- 删除节点调用 `parent_window.canvas.remove_node_with_cleanup()`

### 注册到 NodeCanvas

在 `NodeCanvas.__init__` 中连接信号：
```python
# 在 add_node_to_canvas 中添加：
node.expand_requested.connect(self.on_node_expand_requested)

# 新增方法
def on_node_expand_requested(self, node_name):
    from ui.panels.node_expand_panel import NodeExpandPanel
    panel = NodeExpandPanel(node_name, self.parent_window)
    panel.exec()
```

同时在右键菜单的「节点配置」下方增加「展开节点」项。

## 3. GUI 日志输出从 print 迁移到 logging

### 现状

项目中 **9 个文件** 共约 **211 处** `print()` 调用，无任何 `logging` 模块使用。

### 方案

#### 3.1 新建日志配置模块

**文件**：`ui/core/logger.py`

```python
import logging
import sys
from pathlib import Path

def setup_logger(name: str = "BNOS") -> logging.Logger:
    """配置并返回全局 logger"""
    logger = logging.getLogger(name)
    logger.setLevel(logging.DEBUG)

    # 控制台输出（INFO 级别）
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setLevel(logging.INFO)
    console_fmt = logging.Formatter(
        '[%(asctime)s] %(levelname)-5s %(name)s: %(message)s',
        datefmt='%H:%M:%S'
    )
    console_handler.setFormatter(console_fmt)
    logger.addHandler(console_handler)

    # 文件输出（DEBUG 级别，保留完整日志）
    log_dir = Path(__file__).parent.parent.parent / "logs"
    log_dir.mkdir(exist_ok=True)
    file_handler = logging.FileHandler(
        log_dir / "bnos_gui.log", encoding='utf-8'
    )
    file_handler.setLevel(logging.DEBUG)
    file_fmt = logging.Formatter(
        '[%(asctime)s] %(levelname)-5s %(name)s (%(filename)s:%(lineno)d): %(message)s',
        datefmt='%Y-%m-%d %H:%M:%S'
    )
    file_handler.setFormatter(file_fmt)
    logger.addHandler(file_handler)

    return logger


# 全局 logger 实例
logger = setup_logger()
```

#### 3.2 迁移策略

| 级别 | 原 print() | 新 logger 调用 |
|------|-----------|----------------|
| `DEBUG` | 详细调试信息（如路径打印、状态检查） | `logger.debug()` |
| `INFO` | 正常操作信息（如节点加载、状态变更） | `logger.info()` |
| `WARNING` | 可恢复的问题（如配置未找到、节点创建跳过） | `logger.warning()` |
| `ERROR` | 异常和错误 | `logger.error()` |

#### 3.3 迁移顺序（按文件）

1. `ui/core/logger.py` — 新建
2. `ui/creators/node_creator_manager.py` — 25 处
3. `ui/menu/menu_manager.py` — 2 处
4. `ui/panels/node_group_manager.py` — 14 处
5. `ui/canvas/items/node_item.py` — 2 处
6. `ui/canvas/items/edge_item.py` — 5 处
7. `ui/panels/node_list_panel.py` — 49 处
8. `ui/panels/property_panel.py` — 25 处
9. `ui/canvas/canvas_view.py` — 67 处（最多）
10. `ui/main_window.py` — 47 处
11. `bnos_gui.py` — 入口处初始化 logger

#### 3.4 使用示例

```python
# 旧代码
print(f"✅ 节点 '{node_name}' 已添加到画布")

# 新代码
from ui.core.logger import logger
logger.info("节点 '%s' 已添加到画布", node_name)
```

> **注意**：`logger.info()` 使用 `%s` 格式化而非 f-string，以便延迟求值（性能优化）。

## 重点难点

1. **节点展开组件**：需要处理好 `output.json` 文件监听和编辑锁定，右侧按钮需根据节点运行状态动态切换
2. **节点监测实时更新**：`QTimer` + `mtime` 方案比 `QFileSystemWatcher` 更可靠，后者在某些平台上对频繁写入支持不佳
3. **log 迁移量大**：211 处改动不可一次完成，建议按文件分批次提交
4. **FloatingPanel 基类**：需要确保 `NodeListPanel` 和 `NodeConfigDialog` 的现有 behavior（拖动、位置跟随、最小化）不退化
