# 下一步计划

---

## 1. 修复 JSON 启动无法启动虚拟环境问题

当前通过 JSON 配置启动节点时，无法正确激活虚拟环境，导致节点启动失败。需要修复启动流程，确保 JSON 启动方式能正确识别并激活节点的独立 venv。

---

## 2. 开发 Rust 节点 JSON 启动功能

目前 Rust 节点的 JSON 启动功能尚未实现，需要开发对应的启动逻辑，使 Rust 节点能够像 Python 节点一样通过 JSON 配置进行启停管理。

---

## 3. 开发画布节点直接显示节点运行状态功能

在画布上的节点本体上直接展示节点的实时运行状态信息（如 CPU 占用、内存使用、运行时长等），无需切换到监测面板，提升用户监控效率。
