# BNOS 下一步计划 — 全局字符串国际化改造

> 状态：方案阶段

---

## 目标

对项目所有 `.py` 文件中面向用户展示的字符串进行集中管理，为后续多语言支持打基础。

---

## 改造范围与规则

### ✅ 需要替换的字符串（用户可见）

| 类别 | 示例 |
|------|------|
| 界面显示 | 节点列表标题、组名、占位文本 |
| 日志打印 | `logger.info("加载节点: ...")` |
| 弹窗提示 | `QMessageBox.information(self, "成功", ...)` |
| Toast 消息 | `show_toast("已创建项目", "success")` |
| QAction 文本 | `QAction("新建项目", ...)` |
| 窗口标题 | `setWindowTitle("节点列表")` |
| 按钮文字 | `QPushButton("启动节点")` |
| 状态文本 | `setToolTip("点击展开")` |
| 菜单项 | `menu.addAction("刷新列表")` |
| 错误提示 | `QMessageBox.critical(self, "错误", ...)` |

### ❌ 绝对不替换

| 类别 | 示例 |
|------|------|
| 文件路径 | `"nodes/"`, `"config.json"`, `os.path.join(...)` |
| 变量名 | 所有标识符 |
| JSON key | `"node_name"`, `"listen_upper_file"` |
| 正则表达式 | `r"/nodes/([^/]+)"` |
| SQL 语句 | 无 |
| 颜色十六进制码 | `"#1e1e1e"`, `"#007acc"` |
| 数字字符串 | `"0"`, `"1"`, `"500"` |
| 配置字段名 | `"canvas_size"`, `"view_state"` |
| 内部协议字段 | `"type"`, `"name"`, `"nodes"` |
| f-string 模板 | `f"节点 {name} 已启动"` ← 保持原样 |
| 多行字符串/长文本 | 超过 2 行的字符串不强行拆分 |

---

## 改造方案

### 1. 统一字符串函数 `t()`

```python
# ui/core/i18n.py
STRINGS = {}  # 由 strings_cn.json 加载

def t(key: str) -> str:
    return STRINGS.get(key, key)
```

### 2. 替换模式

```
改造前:
    show_toast("项目已创建", "success")
    QAction("新建项目", main_window)

改造后:
    show_toast(t("k_project_created"), "success")
    QAction(t("k_new_project"), main_window)
```

### 3. Key 命名规范

- 前缀 `k_`，下划线小写
- 语义清晰，见名知义
- 按功能模块分组前缀：

| 前缀 | 含义 | 示例 |
|------|------|------|
| `k_project_*` | 项目管理 | `k_project_open`, `k_project_new` |
| `k_node_*` | 节点操作 | `k_node_create`, `k_node_start` |
| `k_canvas_*` | 画布操作 | `k_canvas_clear`, `k_canvas_zoom` |
| `k_menu_*` | 菜单项 | `k_menu_file`, `k_menu_edit` |
| `k_confirm_*` | 确认对话框 | `k_confirm_delete_node` |
| `k_error_*` | 错误提示 | `k_error_no_project` |
| `k_title_*` | 窗口标题 | `k_title_node_list` |
| `k_label_*` | 标签/占位 | `k_label_project_path` |

### 4. strings_cn.json 结构

```json
{
  "k_project_new": "新建项目",
  "k_project_open": "打开项目",
  "k_project_created": "项目已创建",
  "k_node_create": "新建节点",
  "k_node_start": "启动节点",
  ...
}
```

## 注意事项

1. **不改逻辑**：只替换字面量字符串，不动控制流、函数签名、数据结构
2. **不碰 f-string**：`f"节点 {name}"` 保持原样（格式化逻辑暂不纳入）
3. **不碰注释**：代码注释中的中文不替换
4. **logger 分类处理**：只替换用户可见的 logger 消息，纯调试/开发日志不替换
5. **QMessageBox 标题**：`"成功"`, `"错误"`, `"警告"` 这类常量也纳入替换
6. **不拆分多行**：超过 2 行的长字符串保持原地，不做 key 化
