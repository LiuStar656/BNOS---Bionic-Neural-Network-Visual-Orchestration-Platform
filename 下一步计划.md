# BNOS 绘图工具开发落地方案

> 状态：待实施 | 适配进程隔离架构，画布进程内部实现，不改现有节点逻辑

---

## 一、整体架构定位

### 进程划分

| 进程 | 职责 |
|------|------|
| 主进程 | IPC 调度、菜单控制、全局状态管理，不参与绘图渲染 |
| 画布进程 | 承载**节点层 + 绘图层 + 网格层**三层渲染，崩溃自动重启 |
| 面板进程 | 节点列表、属性面板、监测面板 |
| 核心进程 | 后台业务逻辑 |

节点间维持原有 **JSON 增量文件通信**，绘图功能与节点通信逻辑完全隔离。

### 图层 Z 轴顺序（固定不变）

```
顶层：节点 + 锚点 + 连线   （核心交互层，优先级最高）
中层：图形标注绘图层        （矩形、多边形、箭头、文字）
底层：网格背景层
```

## 二、绘图工具功能清单

### 工具集（纯结构化图形）

| 工具 | 特性 |
|------|------|
| 矩形工具 | 填充 + 描边 |
| 圆角矩形工具 | 填充 + 描边 |
| 折线多边形工具 | 单击加点，双击闭合，支持填充 |
| 直线/箭头工具 | 仅描边 |
| 文本工具 | 背景填充 |

### 绘图层控制

- **锁定绘图层**：屏蔽所有图形交互，保证节点操作不受干扰
- **显示/隐藏绘图层**：一键切换，专注节点编辑
- **独立撤销/重做**：图形操作历史独立管理

### 图形通用交互

- 单击选中、拖拽移动、控制点缩放
- 框选 + 批量移动/删除
- 右键：复制、剪切、粘贴、删除

### 统一样式面板

- 描边颜色/粗细 | 填充颜色/透明度 | 文字大小/颜色

### 交互优先级

- **默认模式**：鼠标优先选中节点/连线，图形仅展示不干扰
- **按住 Alt**：切换为图形编辑模式，可选中修改图形

---

## 三、IPC 指令（主进程 ↔ 画布进程）

### 主进程下发

- 切换绘图工具（矩形/箭头/文字…）
- 设置全局样式（颜色/粗细/透明度）
- 锁定/解锁绘图层
- 显示/隐藏绘图层
- 撤销/重做

### 画布进程回传

- 图形数据变更 → 同步完整图形列表
- 画布进程异常崩溃信号 → 触发自动重启

## 四、数据持久化

图形数据存入 `canvas_layout.json`，与节点布局合并存储：

```json
"graphics": [
    {"type": "rect",    "points": [x1,y1,x2,y2], "style": {...}},
    {"type": "polygon", "points": [x1,y1,...],     "style": {...}},
    {"type": "arrow",   "points": [x1,y1,x2,y2], "style": {...}},
    {"type": "text",    "text": "...", "x": 0, "y": 0, "style": {...}}
]
```

画布进程启动时自动加载，修改后自动保存。

## 五、模块拆分（≤500 行/文件）

| 文件 | 职责 | 预估 |
|------|------|------|
| `ui/canvas/draw_layer.py` | 统一管理图形渲染/选中/拖拽/编辑，撤销重做栈，Alt 切换 | ≤500 |
| `ui/canvas/graphic_items.py` | 图形基类 + 矩形/多边形/箭头/文本子类 | ≤400 |
| `ui/canvas/draw_toolbar.py` | 绘图工具栏（工具切换 + 样式面板 + 图层控制） | ≤300 |
| `ui/canvas/canvas_view.py` | 集成绘图层渲染调用（不改现有节点逻辑） | 增量 |

## 七、设计准则

1. **不改现有架构**：节点渲染/通信逻辑一行不动，只新增绘图层模块
2. **≤500 行/文件**：AI 开发友好，出错率低
3. **新增功能只增文件**：不修改 `node_item.py`、`edge_item.py` 等核心文件
4. **进程隔离**：绘图崩溃 = 画布进程重启，主进程+节点系统不受影响
