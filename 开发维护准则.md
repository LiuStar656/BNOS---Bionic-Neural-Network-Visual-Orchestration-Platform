> 基于 2026-05-21 全面代码扫描和重构经验总结

## 1. 文件规模控制

**铁律**：单个 `.py` 文件不超过 800 行。超过 500 行必须规划拆分。

| 文件 | 当前行数 | 状态 |
|------|---------|------|
| `ui/panels/node_list_panel.py` | 1659 | 待拆分 |
| `ui/panels/property_panel.py` | 1298 | 待拆分 |
| `ui/canvas/canvas_view.py` | 1185 | 已有 4 个 Mixin，可继续按功能拆分 |
| `ui/main_window.py` | 935 | 已降至目标范围 |
| `tools/rust_create_node.py` | 1083 | 工具脚本，暂不限制 |

**拆分策略**：
- 相同职责的方法组提取为 **Mixin** 类（如 `CanvasColorsMixin`）
- 纯函数工具提取为**独立模块**（如 `node_process.py`）
- 新功能一律用 Mixin 或独立模块，不改动现有大类

## 2. 代码重复零容忍

### 原则
任何逻辑出现第 2 次就必须提取为函数，**绝不允许出现第 3 次**。

### 当前已知的重复（待修复）

| 重复逻辑 | 出现次数 | 位置 | 方案 |
|----------|---------|------|------|
| 进程终止 (taskkill/killpg) | 3 次 | `node_process.py`, `node_list_panel.py`, `property_panel.py` | 统一委托 `stop_node_process()` |
| 打开文件夹 (platform.system) | 3 次 | `node_list_panel.py`, `node_monitor.py`, `property_panel.py` | → `ui/core/platform_utils.py` |
| save_config (json.dump) | 4 次 | `canvas_colors.py`, `canvas_view.py`×3 | → 公共 `_save_config(node_info)` |
| 路径规范化 (abspath/normpath) | 2 次 | `node_list_panel.py`, `property_panel.py` | → `ui/core/platform_utils.py` |

## 3. 颜色与尺寸常量集中管理

### 禁止行为
在多个文件的 QSS/styleSheet 中**硬编码相同的颜色值或像素值**。

### 正确做法
- **画布相关颜色**：从 `canvas_view.py` 的属性统一引用
- **主题 QSS**：使用 `theme.py` 中的 `DARK_QSS` 变量
- **公共尺寸**：定义在 `ui/core/constants.py`（待创建）

当前 12 种颜色在 6 个文件中重复定义（`#1e1e1e`、`#cccccc`、`#007acc` 等），`40` 像素偏移在 4 处硬编码。

## 4. Lambda 闭包安全规范

### 问题
Python lambda 闭包存在**延迟绑定**：循环或条件分支中创建的 lambda 在调用时才查找变量值。

### ❌ 错误写法
```python
for lang in ["Python", "Go"]:
    a.triggered.connect(lambda: create_node(lang))  # 总是用最后一个值
```
```python
a.triggered.connect(lambda: self.change_color(item))  # item 可能已变化
```

### ✅ 正确写法
```python
for lang in ["Python", "Go"]:
    a.triggered.connect(lambda ln=lang: create_node(ln))  # 用默认参数捕获
```
```python
a.triggered.connect(lambda item=item: self.change_color(item))
```

**检查方法**：搜索 `lambda` 后面紧跟 `:` 而不是 `=`，人工确认是否有必要加默认参数。

## 5. Logger 使用规范

### ❌ 禁止
```python
logger.info(f"节点 {name} 已启动")        # f-string 立即求值，浪费性能
logger.info("节点 %name 已启动")           # 语法错误，%name 不是合法的格式符
```

### ✅ 正确
```python
logger.info("节点 %s 已启动", name)        # 惰性求值，标准化
logger.info("加载完成: %d个节点, %d条连线", len(self.nodes), len(self.edges))
```

### 已知问题
`node_list_panel.py` 有 **15+ 处** `%var` 语法错误（应写 `%s`），功能上等同于硬编码字符串而无变量值。

## 6. 两层测试流程（必须遵守）

每次代码修改（尤其解耦/重构）必须按顺序执行：

```
步骤1: 创建新模块 → 测试导入
步骤2: 旧代码移除 → 测试导入
步骤3: 启动软件 → 手动验证功能
```

**命令**：
```bat
# 步骤1-2 测试
myenv_new\Scripts\python.exe -c "from ui.xxx import YYY; print('OK')"

# 步骤3 测试
myenv_new\Scripts\python.exe bnos_gui.py
```

**原则**：先建后删，绝不先删后建。

## 7. 循环依赖禁令

### 禁止
```python
# edge_item.py
from ui.canvas.canvas_view import NodeCanvas  # 循环依赖!
```

### 正确
```python
# edge_item.py 已有 self.canvas 引用，直接使用
self.canvas.complete_connection_to_input(self)
```

**检查方法**：`grep -r "from ui\." ui/ | sort` 检查是否有 A→B→A 链。

## 8. 模块组织规范

### 目录职责

| 目录 | 职责 | 文件数上限 |
|------|------|-----------|
| `ui/core/` | 纯功能模块：主题、进程、配置、工具 | 不限 |
| `ui/canvas/` | 画布核心 + Mixin | 不限 |
| `ui/canvas/items/` | 画布图形项（Node/Edge/Anchor） | 不限 |
| `ui/panels/` | 浮动面板和对话框 | 每个文件 ≤800 行 |
| `ui/menu/` | 菜单系统 | 不限 |

### 新增模块命名
- 功能模块：`ui/core/xxx.py`
- 画布 Mixin：`ui/canvas/canvas_xxx.py`
- 面板组件：`ui/panels/xxx.py`

### 类命名
- Mixin 类：`CanvasXxxMixin`
- 面板类：`XxxPanel`
- 对话框：`XxxDialog`

## 9. 批次提交规范

每次提交必须满足：
- 只做**一件事**（一个功能/修复/重构）
- 通过步骤 6 的两层测试
- commit message 格式：`类型(范围): 描述`
  - 类型：`feat` / `fix` / `refactor` / `style` / `perf` / `docs`
  - 示例：`refactor(p3): 提取画布颜色到canvas_colors Mixin`
