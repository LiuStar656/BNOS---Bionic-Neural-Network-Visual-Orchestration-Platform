# 更新日志

## 2026-06-06

### 功能修复

1. **重启功能异常修复**
   - 修复了系统重启时未显示"节点正在运行是否关闭"确认提示对话框的问题
   - 在`_restart_application`方法中添加了运行中节点检测和确认对话框逻辑
   - 确保用户可以选择关闭运行中的节点或让它们在后台继续运行

2. **数据持久化问题修复**
   - 优化了画布布局保存逻辑，确保节点位置、尺寸、样式等信息正确保存
   - 增强了布局加载时的错误处理和数据恢复能力
   - 修复了节点样式和尺寸不匹配的问题

3. **状态同步故障修复**
   - 修复了节点启动后状态信息未实时更新的问题
   - 确保节点监控模块正确获取并显示CPU、内存、运行时长等状态信息
   - 优化了状态更新信号的传递和处理机制

4. **UI布局缺陷修复**
   - 调整了节点内部元素的间距，增强了视觉体验和操作便捷性
   - 优化了节点名称、状态灯、语言标签等元素的位置
   - 调整了状态显示组件的布局，避免元素重叠

### 代码优化

1. **节点监控模块优化**
   - 重构了NodeMonitor类，使用QTimer替代线程进行定时更新
   - 优化了状态更新逻辑，减少了不必要的计算
   - 增强了错误处理和恢复能力

2. **代码结构优化**
   - 将布局保存/加载逻辑移至CanvasLayoutMixin
   - 将颜色设置方法移至CanvasColorsMixin
   - 提高了代码的模块化程度和可维护性

### 其他改进

1. **自动保存机制优化**
   - 增加了防抖机制，避免频繁保存
   - 优化了保存时机，确保布局变化及时保存

2. **日志系统优化**
   - 增加了更多详细的日志信息
   - 优化了日志格式，提高了可读性

3. **错误处理增强**
   - 增强了各种异常情况的处理
   - 提供了更友好的错误提示信息
