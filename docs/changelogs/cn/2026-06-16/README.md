## 📋 本次输出文档

### 1. 卡顿问题迭代优化方案
- **文件路径**：`docs/changelogs/cn/2026-06-16/01_卡顿问题迭代优化方案.md`
- **内容摘要**：
  - **问题范围**：打开项目卡顿、加载画布卡顿、多项目切换卡顿
  - **瓶颈诊断**：
    - 渲染层：`FullViewportUpdate` 过度重绘、`NoCache` 节点无像素缓存、`scene().update()` 全场景刷新风暴
    - IO 层：`project_refresh` 在主线程同步扫描磁盘、`load_layout` 多次遍历节点/连线
    - 算法层：切换项目时重复扫描磁盘（数据已在内存）
    - 后台噪声：PollingManager 1000ms 主定时器过于频繁
  - **核心方案**（4 轮迭代）：
    - **迭代 1（P0）**：渲染层小改动 — `SmartViewportUpdate`、`DeviceCoordinateCache`、`self.update()` 替代全场景刷新
    - **迭代 2（P0）**：异步化 — 新增 `ProjectLoadWorker(QThread)`，`project_open/project_refresh` 走 Signal/Worker 模式
    - **迭代 3（P1）**：算法层 — `load_layout` 合并遍历、切换项目跳过重新扫描、`update_node_list` 增量更新、`app_config.save()` 防抖
    - **迭代 4（P2）**：后台降噪 — PollingManager 降频
  - **量化目标**：主线程阻塞时间降低 70%+，空闲 CPU ≤1%
  - **风险评估**：R-1 ~ R-6 六项风险，每项均有回滚方案
  - **验证清单**：功能回归测试 12 项 + 性能对比测试 6 项 + 日志检查 3 项

## 📌 实施路线图

```
阶段 1：渲染层（30 分钟）   →  迭代 1：最小改动，最大收益
阶段 2：异步化（1~2 小时）   →  迭代 2：把磁盘 IO 移出主线程
阶段 3：算法层（1~2 小时）   →  迭代 3：消除重复工作
阶段 4：后台降噪（30 分钟）  →  迭代 4：降低空闲 CPU 噪声
```
