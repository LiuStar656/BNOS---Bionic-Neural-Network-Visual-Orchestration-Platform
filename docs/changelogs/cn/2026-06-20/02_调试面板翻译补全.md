# 02_调试面板翻译补全

**日期**: 2026-06-20

## 背景

调试面板（`DebugPanel`）是 2026-06-18 新增的浮动面板，用于管理节点调试会话。但由于翻译键（k 值）未在上游 `strings_cn.json` / `strings_en.json` 中定义，面板中多个标签直接显示原始键名而非用户可读文本。

**典型症状**：
- 标签显示 `k_port` 而非「端口」
- 按钮显示 `k_start` 而非「启动」
- 列头显示 `k_mode` 而非「模式」

## 变更内容

### 新增翻译键

在 `strings_cn.json` 和 `strings_en.json` 中补全以下 17 个翻译键：

| 键名 | 中文 | English | 用途 |
|------|------|---------|------|
| `k_port` | 端口 | Port | 调试端口显示 |
| `k_mode` | 模式 | Mode | 调试模式选择 |
| `k_pattern` | 模式 | Pattern | 日志匹配模式 |
| `k_action` | 操作 | Action | 操作列头 |
| `k_value` | 值 | Value | 值显示 |
| `k_start` | 启动 | Start | 启动调试按钮 |
| `k_stop` | 停止 | Stop | 停止调试按钮 |
| `k_pause` | 暂停 | Pause | 暂停调试按钮 |
| `k_resume` | 恢复 | Resume | 恢复调试按钮 |
| `k_breakpoints` | 断点 | Breakpoints | 断点列表 |
| `k_sessions` | 会话 | Sessions | 调试会话列表 |
| `k_log_pattern` | 日志模式 | Log Pattern | 日志过滤模式 |
| `k_add` | 添加 | Add | 添加操作 |
| `k_hits` | 命中 | Hits | 断点命中计数 |
| `k_clear_all` | 全部清除 | Clear All | 清除所有断点 |
| `k_logs` | 日志 | Logs | 日志面板 |
| `k_auto_scroll` | 自动滚动 | Auto Scroll | 日志自动滚动 |
| `k_variables` | 变量 | Variables | 变量查看面板 |

## 影响范围

- **修改文件**: `ui/core/strings_cn.json`, `ui/core/strings_en.json`
- **新增键数**: 18 个（17 个调试面板键 + 1 个 `k_refresh` 补全）
- **无需改代码**: 调试面板代码中已使用 `t("k_xxx")` 调用，补齐翻译文件即生效
