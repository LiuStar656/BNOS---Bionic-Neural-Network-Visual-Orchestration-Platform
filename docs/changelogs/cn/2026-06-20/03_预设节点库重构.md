# 03_预设节点库重构

**日期**: 2026-06-20

## 背景

### 旧模板系统的核心缺陷

原有的"模板选择器"系统存在三个不可调和的矛盾：

1. **保存内容残缺**：仅保存 `config.json` 的字段描述，不保存节点主程序代码（`main.py`）。模板本质上只是一个参数骨架。

2. **应用无意义**：选模板后调用 `node_creator_manager.create_node` 创建**空壳节点**——有 venv、有 config.json、有启动脚本，但 `main.py` 是空的。用户必须重新编写全部业务代码。

3. **与导出功能重叠**：项目已有完整的 `.bnos` 节点导出/导入机制（`ImportExportManager` + `Packager`），可打包完整节点（含代码 + venv + config）。模板系统的价值极低，属于功能重复。

### 用户反馈

- "节点还要和背后的主程序绑在一起不是光有 json 就行的"
- "我已经有导出完整节点的功能了这个模版选择器是不是多此一举？"

## 变更内容

### 架构决策：废弃模板，统一为预设节点库

核心思路：**保存时 = 导出完整节点到 `node_templates/`，导入时 = 从 `node_templates/` 解压到项目 `nodes/`**。所有操作统一复用 `Packager` 的 `.bnos` 打包/解压机制。

## 数据流

```
右键节点 → ActionFactory → _template.py::save_as_preset
  ├── themed_input_dialog（用户输入描述）
  ├── Packager.compress_directory → node_templates/{name}.bnos
  └── json.dump → node_templates/{name}.json

菜单"预设节点库" → PresetLibraryDialog
  ├── _load_presets → 扫描 node_templates/ 目录
  └── _import_preset
      ├── Packager.extract_package → 临时目录
      ├── shutil.move → 当前项目 nodes/{name}
      ├── _repair_portable_venv → 修复可移植 venv
      └── main_window.refresh_nodes → 刷新节点列表
```
