# 节点启动队列功能实现

---

## 更新概述

本次更新实现了完整的节点启动排队系统（V2.0），采用智能调度器设计，具备并发控制、优先级调度、错误重试、启动间隔控制和队列持久化等核心特性。当前仅用于限流，不支持依赖启动。

---

## 核心功能

### 1. 启动队列管理

支持节点进入队列等待启动，提供完整的队列操作接口：

| 接口 | 功能 |
|------|------|
| `enqueue(node_name, priority, dependencies)` | 将节点加入队列 |
| `dequeue(node_name)` | 从队列移除节点 |
| `promote(node_name, priority)` | 提升节点优先级（插队） |
| `clear()` | 清空队列 |
| `get_status(node_name)` | 获取节点排队状态 |
| `get_queue_info()` | 获取队列整体信息 |

### 2. 并发控制

可配置同时启动的节点数量，默认值为 2，避免资源竞争：

```python
startup_queue = NodeStartupQueueManager(
    max_concurrent=2,   # 最大并发数
    max_retry=3,        # 最大重试次数
    batch_delay=300     # 批次间隔（毫秒）
)
```

### 4. 状态反馈

新增排队状态，完善节点状态体系：

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| `queued` | ◎ | blue | 排队中 |
| `starting` | ◐ | yellow | 启动中 |
| `running/idle` | ● | green | 运行中/空闲 |
| `failed` | ✗ | red | 启动失败 |

### 5. 错误重试

支持失败自动重试机制，可配置重试次数（默认 3 次）：
- 启动失败时自动将节点状态改回 `QUEUED`
- 等待配置的重试延迟后重新调度
- 达到最大重试次数后标记为 `FAILED`

### 6. 启动间隔控制

批次间平滑延迟，避免资源抢占：
- 使用 `QTimer.singleShot` 递归调度
- 相邻批次之间加入 200ms~500ms 的间隔
- 给系统留出资源释放和状态同步时间

### 7. 队列持久化

退出时保存队列状态，重启后恢复：

**快照管理**：
- `save_snapshot()`：将队列状态保存到 JSON 文件
- `load_snapshot()`：启动时加载队列快照
- 支持恢复提示：显示上次未完成的队列状态

## 数据结构设计

### QueueItem 类

```python
class QueueItem:
    def __init__(self, node_name: str, priority: int = 0):
        self.node_name = node_name          # 节点名称
        self.priority = priority            # 优先级
        self.status = QueueStatus.QUEUED    # 当前状态
        self.retry_count = 0                # 已重试次数
        self.enqueue_time = None            # 入队时间
        self.start_time = None              # 开始启动时间
        self.complete_time = None           # 完成时间
        self.error_message = None           # 错误信息
```

### QueueStatus 枚举

```python
class QueueStatus(Enum):
    QUEUED = "queued"           # 排队中
    STARTING = "starting"       # 启动中
    SUCCESS = "success"         # 启动成功
    FAILED = "failed"           # 启动失败
    CANCELLED = "cancelled"     # 已取消
```

## 关键实现细节

### 智能队列调度器

```python
def _process_queue(self):
    """队列调度主逻辑 - 智能调度器
    
    核心特性：
    1. 并发控制：同时启动的节点数不超过 max_concurrent
    2. 优先级调度：高优先级节点优先
    3. 平滑启动：批次间加入延迟，避免资源竞争
    """
    # 检查空闲槽位
    running_count = len([item for item in self._queue 
                         if item.status == QueueStatus.STARTING])
    
    if running_count >= self._max_concurrent:
        self._schedule_next_process()
        return
    
    # 获取就绪节点（排队中的节点）
    ready_items = self._get_ready_nodes()
    
    # 按优先级排序
    ready_items.sort(key=lambda x: (-x.priority, x.enqueue_time))
    
    # 启动下一个就绪节点
    next_item = ready_items[0]
    self._start_node(next_item)
    
    # 调度下一次处理（带延迟）
    self._schedule_next_process()
```

## 配置与默认值

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `max_concurrent` | 2 | 最大同时启动的节点数 |
| `max_retry` | 3 | 失败重试次数 |
| `retry_delay` | 5000 | 重试延迟（毫秒） |
| `batch_delay` | 300 | 批次启动间隔（毫秒） |
| `queue_timeout` | 60000 | 队列超时时间（毫秒） |
