# 多锚点系统完善与连线持久化

## 📋 更新概述

本次更新完善了面板模式下的多锚点输入系统，解决了锚点定位、端口映射、连线持久化和批量清理等关键问题。主输入锚点（`listen_upper_file`）与附加输入端口（`input_port`）在尺寸、位置和配置写入上完全区分，重启后连线端口信息不再丢失。

---

## 🎯 核心变更

### 1. 锚点差异化设计

**两种锚点类型**：

| 锚点 | 尺寸 | 位置 | config 写入字段 |
|------|------|------|----------------|
| `listen_upper_file`（主输入） | 16px | 左边界垂直居中 | `listen_upper_file` |
| `input_port`（附加输入） | 10px | 紧贴标签左侧（x=21） | `port_mappings[port_name]` |

**位置对比**：

```
旧布局（全部一样大，都在左边界）：
  [● 16px]  listen_upper_file
  [● 16px]  prompt
  [● 16px]  context

新布局（尺寸区分，小锚点贴近标签）：
  [● 16px]  listen_upper_file（左边界居中）
  [○ 10px]  prompt（标签左侧 2px 间隙）
  [○ 10px]  context（标签左侧 2px 间隙）
```

**修改文件**：
- `ui/canvas/items/anchor_item.py`：新增 `ANCHOR_SIZE_SMALL=10`、`ANCHOR_HALF_SMALL=5`，构造函数接受 `size` 参数，hover 效果根据尺寸动态适配
- `ui/canvas/items/node_item.py`：`_build_detailed_view` 中 `_param_row_positions` 改为 `(center_x, center_y, size)` 三元组，小锚点 x = `ANCHOR_ZONE_WIDTH + LEFT_INNER_PADDING - ANCHOR_SIZE_SMALL/2 - 2 = 21`
- `ui/canvas/items/anchor_manager.py`：`_make_anchor` 接受 `size` 参数，`_find_nearest` 根据实际尺寸计算中心距离，`build_from_config` 区分默认锚点（16px）与附加端口锚点（10px）

---

### 2. config.json 端口映射修正

**问题**：连到 `listen_upper_file` 锚点的路径被写入 `port_mappings["default"]`，而正文字段 `listen_upper_file` 仍为空或保持模板默认值。

**修复**：`canvas_connections.py` 的 `create_edge` 和 `remove_edge` 中增加端口名判断：

```python
# port_name == "default" 或 None → 读写 listen_upper_file
# port_name == "prompt"/"context"... → 读写 port_mappings[port_name]
if port_name and port_name != "default":
    target_config['port_mappings'][port_name] = source_output_path
else:
    target_config['listen_upper_file'] = source_output_path
```

**修改文件**：`ui/canvas/canvas_connections.py`（`create_edge` L132-143, `remove_edge` L204-219）

---

### 3. 连线层级提升

**问题**：线条 z-value 为 0，被节点（z=2）和锚点（z=10）遮挡，视觉效果不佳。

**修复**：EdgeItem 层级从 0 提升到 20，成为画布最顶层。

**层级关系**：`节点(z=2) < 锚点(z=10) < 线条(z=20)`

**修改文件**：`ui/canvas/items/edge_item.py`（L65）

---

### 4. 连线持久化修复

**问题**：重启后所有连线重新绑定到默认 `listen_upper_file` 锚点，小锚点（如 `prompt`、`context`）的连接信息丢失。

**根因**：
1. `save_layout` 不保存端口名
2. `load_layout` 创建 EdgeItem 时不传递端口信息
3. `_validate_edge_anchor_binding` 使用 `or` 条件，一端未绑定会导致另一端也被覆盖为默认锚点

**修复**：

| 位置 | 修改 |
|------|------|
| `save_layout` | edge dict 增加 `source_port` / `target_port` 字段 |
| `load_layout` | 根据端口名从 `anchor_manager.get_input/output()` 恢复绑定 |
| `_validate_edge_anchor_binding` | `if not a or not b` → 逐锚点独立检查 `if not a` / `if not b` |

**持久化格式变化**：
```json
// 之前
{"source": "node_rust_1", "target": "node_python_1"}

// 之后
{"source": "node_rust_1", "target": "node_python_1", "target_port": "prompt"}
```

**修改文件**：`ui/canvas/canvas_layout.py`

---

### 5. 批量清除完善

**问题**：`clear_canvas` 直接调 `edge.remove_from_scene()` 跳过 config 清理；`batch_clear_listen_config` 只清 `listen_upper_file` 不处理 `port_mappings`。

**修复**：

| 方法 | 修改 |
|------|------|
| `clear_canvas` | `edge.remove_from_scene()` → `self.remove_edge(edge)`，走完整 config 清理 |
| `batch_clear_listen_config` | 同步清除 `listen_upper_file` + `port_mappings` + `out_connections` |

**修改文件**：
- `ui/canvas/canvas_view.py`（L964-968）
- `ui/canvas/canvas_batch_ops.py`（L123-154）
