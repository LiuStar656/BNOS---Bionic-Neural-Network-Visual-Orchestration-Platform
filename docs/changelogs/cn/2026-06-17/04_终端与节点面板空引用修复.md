# 终端 Dock 与节点面板空引用修复（AttributeError 全消除）

---

## 问题 1：`_terminal_dock` 未初始化即被访问

### 现象

```
Traceback (most recent call last):
  File "ui/core/canvas_host.py", line 633, in _update_terminal_on_canvas_change
    if hasattr(self._terminal_dock, 'sync_to_canvas'):
               ^^^^^^^^^^^^^^^^^^^
AttributeError: 'CanvasHost' object has no attribute '_terminal_dock'. Did you mean: '_init_terminal_dock'?
```

### 根因分析

#### CanvasHost 的终端 dock 初始化时机

```
# CanvasHost 初始化流程
# ───────────────────────────────────────────────────────

  t=0  __init__()
        ├─ 创建空白缓冲层 _blank_placeholder
        ├─ 设置样式
        │
        │   此时：self._terminal_dock 不存在！
        │
        └─ 等待外部调用...

  t=1  add_canvas_dock(project_name, project_path)  ← 第一个画布创建
        └─ _create_canvas_dock()
             ├─ 创建 canvas (NodeCanvas)
             ├─ 创建 canvas_dock (QDockWidget)
             ├─ canvas_dock.setWidget(canvas)
             └─ ✅ len(self._canvas_docks) == 1 → 调用 _init_terminal_dock()
                    └─ self._terminal_dock = TerminalDock(...)
                          ↑
                       只有在"第一个画布创建后"才会初始化
```

#### `_update_terminal_on_canvas_change` 的调用时机

```
# 画布切换流程
# ───────────────────────────────────────────────────────

  当用户切换画布 tab 或 新画布被创建时：

  canvas_changed.emit(canvas)   ← 信号发射
       └─ _update_terminal_on_canvas_change(canvas)   ← 这个方法被调用
              │
              └─ hasattr(self._terminal_dock, 'sync_to_canvas')
                     ↑
                 ❌ 如果这是第一个画布，且 _init_terminal_dock()
                    还没被调用 → self._terminal_dock 不存在！
```

#### 触发条件

1. 启动应用自动打开项目
2. 第一个画布 dock 正在创建 → `_canvas_docks` 变成 `[dock]`
3. 创建完成后触发 `canvas_changed` 信号 → 调用 `_update_terminal_on_canvas_change`
4. **但**：`_init_terminal_dock()` 可能在信号发射时还**未执行完**（取决于 Qt 的信号队列）
5. → `self._terminal_dock` 属性不存在 → `AttributeError`

**更简单的解释**：`_update_terminal_on_canvas_change` 试图访问一个"还没创建"的属性。

### 修复方案

#### 修复 1-1：在 `_update_terminal_on_canvas_change` 中检查属性存在性

```python
# ui/core/canvas_host.py

def _update_terminal_on_canvas_change(self, canvas):
    """画布改变时更新终端"""
    # ✅ 先检查终端是否已初始化
    # （_terminal_dock 只在第一个画布创建后才会被初始化）
    if not hasattr(self, '_terminal_dock') or self._terminal_dock is None:
        return
    
    if hasattr(self._terminal_dock, 'sync_to_canvas'):
        self._terminal_dock.sync_to_canvas()
```

#### 修复 1-2（可选）：在 `CanvasHost.__init__` 中预初始化

```python
# ui/core/canvas_host.py

def __init__(self, parent=None):
    super().__init__(parent)
    self._parent_window = parent
    
    # 预初始化（确保 hasattr 检查始终返回 True）
    self._terminal_dock = None   # ← 可选的预防性初始化
    
    # ... 其余初始化
```

**说明**：本次修复采用修复 1-1 即可，修复 1-2 作为额外保险。

## 问题 2：`NodeListDockPanel` 没有 `refresh()` 方法

### 现象

```
Traceback (most recent call last):
  File "ui/main_window/state.py", line 310, in _on_finished
    self.node_list_panel.refresh()
    ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
AttributeError: 'NodeListDockPanel' object has no attribute 'refresh'
```

### 根因分析

#### NodeListDockPanel 的实际刷新方法

```python
# ui/panels/node_list_dock.py

class NodeListDockPanel(QWidget, NodeListOperationsMixin, ...):
    """节点列表面板（Dock 版本）"""
    
    def update_node_list(self, nodes_data):
        """更新节点列表（树形结构，支持分组）"""
        # ✅ 这才是实际的"刷新"方法！
        # - 清空 node_tree
        # - 按分组添加节点
        # - 更新路径显示
        self.nodes_data = nodes_data
        # ...
        self.path_label.setText(...)
    
    # ❌ 没有 refresh() 方法！
```

#### 是谁在调用 `refresh()`？

```python
# ui/main_window/state.py - _auto_open_project 的 _on_finished 回调

def _on_finished(nodes_data, ...):
    ...
    # 6. 同步节点列表面板显示
    if hasattr(self, 'node_list_panel') and self.node_list_panel:
        self.node_list_panel.refresh()   # ❌ 这个方法不存在！
```

**问题**：在本次重构 `_auto_open_project` 时，我添加了"刷新节点列表面板"的逻辑，但误调用了不存在的方法名 `refresh()`。

**为什么之前没有这个问题？**：因为之前 `_auto_open_project` 采用同步路径（`project_refresh(async_mode=False)`），并且**没有**显式调用节点面板刷新。

### 修复方案

#### 修复 2-1：新增 `NodeListDockPanel.refresh()` 便捷方法

```python
# ui/panels/node_list_dock.py

class NodeListDockPanel(...):
    # ...
    
    def refresh(self):
        """便捷刷新方法：从 parent_window.nodes_data 重新加载节点列表
        
        这个方法使 NodeListDockPanel 的接口与常见的刷新语义保持一致，
        同时支持从外部（如 _auto_open_project 的回调中）触发刷新。
        """
        if self.parent_window and hasattr(self.parent_window, 'nodes_data'):
            self.update_node_list(self.parent_window.nodes_data)
```

#### 修复 2-2：在调用处做双重检查

```python
# ui/main_window/state.py - _on_finished 回调

# 6. 同步节点列表面板显示
if hasattr(self, 'node_list_panel') and self.node_list_panel:
    # ✅ NodeListDockPanel 使用 update_node_list(nodes_data) 刷新显示
    if hasattr(self.node_list_panel, 'update_node_list'):
        self.node_list_panel.update_node_list(self.nodes_data)
    # 或者也可以使用新增的 refresh()
    # self.node_list_panel.refresh()
```

**说明**：本次修复同时实现了 2-1 和 2-2，双重保险。

## 关键设计决策

### 决策 1：在 `_update_terminal_on_canvas_change` 中检查 `hasattr` 而非预先初始化
- **理由**：显式检查比隐式初始化更清晰，调用者可以立即知道"为什么什么都没发生"（通过代码注释理解）
- **影响**：如果后续有新的调用者访问 `_terminal_dock`，需要加同样的检查（但这不是问题，因为这是标准防御性编程模式）
- **回滚点**：可以改为在 `__init__` 中设置 `self._terminal_dock = None`，然后检查 `if self._terminal_dock is not None`

### 决策 2：新增 `refresh()` 便捷方法（而非在调用处硬编码调用 `update_node_list`）
- **理由**：`refresh()` 是更通用的接口名，其他代码也可能使用它；同时保留了 `update_node_list` 作为底层实现
- **影响**：增加了一个便捷方法，但代码更清晰，未来改动只需修改 `refresh()` 的实现
- **回滚点**：如果只希望从调用处硬编码调用，可删除 `refresh()` 方法

### 决策 3：双重保险（修复 2-1 + 修复 2-2）
- **理由**：`state.py` 中即使 `refresh()` 方法已存在，加上 `hasattr` 检查可防止未来 `NodeListDockPanel` 重构时再次出现类似问题
- **影响**：轻微增加代码量，但大大增强了对未来重构的容忍度
- **回滚点**：如果认为冗余，可移除调用处的 `hasattr` 检查
