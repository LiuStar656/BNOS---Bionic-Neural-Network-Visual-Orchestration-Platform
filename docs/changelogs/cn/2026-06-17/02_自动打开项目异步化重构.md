# 自动打开项目异步化重构（与 project_open 统一时序）

---

## 问题描述

### 现象
```
[23:52:23] INFO  (state.py): 自动打开上次项目: C:\Users\Lenovo\2
[23:52:24] INFO  (canvas_host.py): [CanvasHost] 创建画布前检查: parent_window=OK, nodes_data=0个节点
[23:52:24] INFO  (canvas_layout.py): [load_layout] 已读取 canvas_layout.json: 0个节点位置
```

- **关键问题**：`nodes_data=0个节点`
- `add_canvas_dock` 被调用时，Worker 还在后台扫描磁盘
- `load_layout` 从 `canvas_layout.json` 读取的节点为 0（或节点信息为空）
- 用户重启后看到画布是空的

### 用户反馈
> "首次打开软件后画布布局没有正常加载，但是二次打开其他项目后其他项目的画布布局却正常加载"

---

## 根因分析

### 调用链对比

```
# ============================================================
# 路径 A: 用户手动打开项目（project_open） — ✅ 正常
# ============================================================

project_open(path)
  └─ Worker = ProjectLoadWorker(path)    ← 后台线程开始扫描磁盘
       └─ Worker.finished.connect(
           _on_finished(nodes_data, ...)
             │
             ├─ nodes_data.clear()       ← 先填充数据
             ├─ for name, info: nodes_data[name] = info
             │
             └─ add_canvas_dock(...)     ← 后创建画布
                    └─ load_layout()     ← 读取 canvas_layout.json
                                              (此时 nodes_data 已有数据)
```

```
# ============================================================
# 路径 B: 启动时自动打开项目（_auto_open_project） — ❌ 崩溃/空画布
# ============================================================

_auto_open_project(path)   ← 修改前
  ├─ project_refresh(self, async_mode=False)   ← 实际还是异步！
  │     └─ Worker = ProjectLoadWorker(path)   ← Worker 在后台跑...
  │                                                 还没回来
  │
  └─ self._canvas_host.add_canvas_dock(...)     ← ← ← 立即创建画布！
          └─ load_layout()                          此时 nodes_data = {}
                 └─ canvas_layout.json 读取到 0 个节点位置
```

### 问题根源：`project_refresh` 的 `async_mode` 参数被忽略

```python
# ui/core/project_manager.py
#
# 即使传入 async_mode=False，project_refresh 仍然启动异步 Worker
# Worker 的 finished 信号被发射时，_auto_open_project 已经创建了画布
#
# 调用时序（时间从上到下）：
#
#   t=0  _auto_open_project 被调用
#   t=1  project_refresh(async_mode=False) 启动 Worker
#   t=2  add_canvas_dock() 创建画布
#   t=3  load_layout() 读取 canvas_layout.json → 0 个节点
#   t=4  Worker 完成扫描，发射 finished 信号 ← 太晚了！
```

### 为什么第二次打开其他项目正常？

```
# 第二次打开项目时：
#   1. 应用已经在运行
#   2. project_open() 走的是"正确路径 A"
#   3. Worker.finished 回调中先填充 nodes_data，再创建画布
#   4. 画布加载正确
#
# 只有"启动时自动打开上次项目"走的是路径 B（_auto_open_project）
# 这就是为什么"首次打开软件不行，二次打开其他项目可以"
```

## 修复方案

### 核心原则
> **`_auto_open_project` 采用与 `project_open` 完全相同的异步模式**

### 修改内容

#### 修改 1：重构 `_auto_open_project` 方法

```python
# ui/main_window/state.py

def _auto_open_project(self, project_dir):
    """内部方法：自动打开指定项目
    
    使用与 project_open 相同的异步模式：先在后台扫描节点，
    扫描完成后再创建画布并恢复布局。确保 nodes_data 填充后
    才调用 add_canvas_dock。
    """
    # 验证项目有效性...
    
    # ✅ 关键：先重置状态 + 清理旧 dock
    if hasattr(self, '_canvas_host') and self._canvas_host:
        self._canvas_host.remove_canvas_dock_by_path(project_dir)
    
    # ✅ Worker 后台扫描（与 project_open 完全相同）
    from ui.core.project_load_worker import ProjectLoadWorker
    worker = ProjectLoadWorker(project_dir, parent=self)
    
    # ✅ Worker 回调：完成后先填充 nodes_data，再创建画布
    def _on_finished(nodes_data, mounted_nodes, running_nodes):
        # 1. 先填充数据（与 project_open._on_finished 一致）
        self.nodes_data.clear()
        for name, info in nodes_data.items():
            self.nodes_data[name] = info
        
        # 2. 再创建画布（此时 nodes_data 已有数据）
        if hasattr(self, '_canvas_host'):
            self._canvas_host.add_canvas_dock(project_name, project_dir)
            
            # 3. 恢复画布状态（分割条位置等）
            from ui.core.window_state_manager import restore_canvas_host_state
            QTimer.singleShot(200, lambda: restore_canvas_host_state(self))
            
            # 4. 统一 UI 更新
            from ui.core.project_manager import _apply_after_refresh
            _apply_after_refresh(self, running_nodes)
        
        # 5. 保存项目到配置
        self.app_config.set("last_project", self.current_project_path)
        self.app_config.save()
        
        # 6. 刷新节点列表面板
        if hasattr(self, 'node_list_panel') and self.node_list_panel:
            if hasattr(self.node_list_panel, 'update_node_list'):
                self.node_list_panel.update_node_list(self.nodes_data)
    
    # ✅ 失败回调
    def _on_failed(err_msg):
        logger.error(f"[auto_open] 项目扫描失败: {err_msg}")
        self.show_toast(f"打开项目失败: {err_msg}", "error")
    
    # ✅ 连接信号并启动
    worker.progress.connect(_on_progress)
    worker.finished.connect(_on_finished)
    worker.failed.connect(_on_failed)
    worker.start()
```

#### 修改 2：`CanvasHost` 新增 `remove_canvas_dock_by_path`

```python
# ui/core/canvas_host.py

def remove_canvas_dock_by_path(self, project_path):
    """根据项目路径移除对应画布 dock
    
    防止同一项目重复打开时 dock 残留
    """
    if not project_path:
        return
    
    normalized_path = os.path.normpath(os.path.abspath(project_path))
    docks_to_remove = []
    
    # 找到所有匹配的 dock
    for dock in self._canvas_docks:
        content = dock.get_content_widget()
        if isinstance(content, NodeCanvas):
            canvas_data = self.get_canvas_data(content)
            canvas_project_path = canvas_data.get('project_path')
            if canvas_project_path:
                if os.path.normpath(os.path.abspath(canvas_project_path)) == normalized_path:
                    docks_to_remove.append(dock)
    
    # 安全移除
    for dock in docks_to_remove:
        try:
            self.removeDockWidget(dock)
            dock.setParent(None)
            dock.deleteLater()
            if dock in self._canvas_docks:
                self._canvas_docks.remove(dock)
            # 清理 canvas_data_map 中的引用
            for key in list(self._canvas_data_map.keys()):
                try:
                    if key == dock or (hasattr(dock, 'get_content_widget') and
                                       key == dock.get_content_widget()):
                        del self._canvas_data_map[key]
                except Exception:
                    pass
            logger.info(f"[CanvasHost] 已移除画布 dock: {project_path}")
        except Exception as e:
            logger.warning(f"[CanvasHost] 移除画布 dock 失败: {e}")
```

  │

  │
  └─ Worker = ProjectLoadWorker(path)
        │

        │     └─ 扫描 6 个节点
        │     └─ 同步节点注册表

              │
              ├─ add_canvas_dock(...)                ← t=4
              │     └─ load_layout()
              │         └─ canvas_layout.json 读取到 M 个节点 ✓
              │
              ├─ restore_canvas_host_state(self)      ← t=4.2
              ├─ _apply_after_refresh(...)
              └─ node_list_panel.update_node_list(self.nodes_data)

## 关键设计决策

### 决策 1：`_auto_open_project` 与 `project_open` 采用完全相同的模式
- **理由**：两个路径应该行为一致，避免分别维护两套逻辑导致回归
- **影响**：代码略有重复，但逻辑清晰，易于维护
- **回滚点**：如果未来需要进一步解耦，可以把 `_on_finished` 回调提取为独立函数

### 决策 2：新增 `remove_canvas_dock_by_path` 而非在 dock 中做去重判断
- **理由**：显式清理比隐式去重更容易理解和调试；方法名清晰表达意图
- **影响**：新增一个方法，但降低了 dock 管理的复杂度
- **回滚点**：始终可以移除这个方法，改为在 `add_canvas_dock` 内部判断

### 决策 3：在 Worker 回调中保留所有后续 UI 更新（恢复状态、刷新面板等）
- **理由**：必须在 Worker 完成后执行，否则会出现"状态恢复时 nodes_data 还没填充"的同类问题
- **影响**：回调函数较长，但保证了正确的时序
- **回滚点**：如果发现回调函数过于庞大，可拆分为多个辅助函数
