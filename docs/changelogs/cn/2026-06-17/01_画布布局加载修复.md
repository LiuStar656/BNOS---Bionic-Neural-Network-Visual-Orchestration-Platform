# 画布布局加载修复（try/finally 保护 + Scene/Viewport 强制刷新）

---

## 问题描述

### 现象 1：画布上有节点但不显示
- 节点已被 `load_layout` 从 `canvas_layout.json` 读取并通过 `NodeItem` 添加到 `QGraphicsScene`
- 但在 UI 上看不到节点
- 重新打开第二个项目后，画布显示正常

### 现象 2：画布上节点/连线变更后不立即渲染
- 拖动节点、增删连线后，需要点击/缩放操作触发才会刷新
- `QGraphicsView` 的 `setUpdatesEnabled(False)` 在异常情况下没有恢复为 `True`

### 现象 3：节点/连线加载后场景不显示
- `canvas_layout.json` 存在且包含节点位置信息
- 节点数量、位置、尺寸在日志中显示正确
- 但渲染层为空 — `QGraphicsScene` 没有触发 `paintEvent`

---

## 根因分析

### 根因 1：`setUpdatesEnabled(False)` 没有 `finally` 保护

```python
# 修改前
self.setUpdatesEnabled(False)
# ... 加载逻辑（可能抛出异常）
self.setUpdatesEnabled(True)   # 如果中间抛异常，这行不会执行
```

- 如果 `load_layout` 中任何一步抛出异常（例如 `canvas_layout.json` 格式错误、`NodeItem` 构造失败）
- `setUpdatesEnabled(True)` 永远不会被调用
- `QGraphicsView` 处于"禁用更新"状态，**永远不会重绘**

### 根因 2：缺少 `QGraphicsScene.update()` 强制场景刷新

```python
# 修改前：NodeItem 添加到 scene 后只调用 update()
node = NodeItem(...)
self.scene.addItem(node)
# scene 可能缓存了渲染结果，新添加项不会立即触发 paintEvent
```

- `QGraphicsScene.addItem()` 只是把 item 加入索引
- 需要显式调用 `scene.update(sceneRect)` 通知渲染层有新内容
- 这在 `canvas_layout.json` 有大量节点的批量加载场景尤其重要

### 根因 3：`viewport()` 缓存问题

- `QGraphicsView` 的 viewport（`QWidget`）有独立的更新机制
- 即使 `scene.update()` 触发了，如果 viewport 没有重绘，视觉上仍可能为空
- 需要额外调用 `self.viewport().update()`

### 根因 4：画布 dock 没有被显式显示

- `CanvasHost` 移除 `_blank_placeholder` 后，`centralWidget` 被设为 `None`
- Qt 的 `QDockWidget` 需要**有效的中央控件**才能正确停靠和渲染
- 即使 dock 被创建并添加到了布局，没有中央控件也可能导致渲染异常

---

## 修复方案

### 修复 1：`load_layout` 增加 `try/finally` 保护

```python
# ui/canvas/canvas_layout.py

try:
    self.setUpdatesEnabled(False)
    # ... 所有加载逻辑：
    # 1. 读取 canvas_layout.json
    # 2. 创建 NodeItem
    # 3. 创建连线 Edge
    # 4. 恢复视图状态（缩放/滚动）
except json.JSONDecodeError:
    logger.warning("canvas_layout.json 格式错误")
except Exception as e:
    logger.exception(f"load_layout 发生未预期异常: {e}")
finally:
    # 无论成功/失败，必须恢复更新
    self.setUpdatesEnabled(True)
    # 强制场景刷新（确保新添加节点立即渲染）
    if hasattr(self, 'scene') and self.scene is not None:
        self.scene.update(self.scene.sceneRect())
    # 强制 viewport 重绘
    if hasattr(self, 'viewport') and callable(self.viewport):
        self.viewport().update()
```

### 修复 2：`CanvasHost._remove_blank_placeholder` 增加透明中央占位控件

```python
# ui/core/canvas_host.py

# 问题：centralWidget = None 后 dock widget 无法正确停靠
# 修复：创建一个透明的 QWidget 作为中央占位控件
central_placeholder = QWidget(self)
central_placeholder.setStyleSheet("background: transparent;")
central_placeholder.setAttribute(Qt.WidgetAttribute.WA_DontShowOnScreen, False)
central_placeholder.setObjectName("canvas_host_central_placeholder")
self.setCentralWidget(central_placeholder)
```

### 修复 3：画布 dock 显式激活

```python
# ui/core/canvas_host.py - _create_canvas_dock 方法末尾

# 显式显示画布 dock 和 canvas widget
canvas_dock.show()
canvas_dock.raise_()
canvas.show()
canvas.raise_()
canvas.setFocus()
```

### 修复 4：移除"从 nodes_data 自动创建节点"逻辑

```python
# 之前添加的逻辑是错误的：
# 遍历 parent_window.nodes_data，为不在 canvas_layout.json 中的节点自动创建 NodeItem 并排到画布
# 正确行为：画布节点只能来自 canvas_layout.json（用户手动拖入/移动）
# 项目中其他节点只在节点列表面板中显示，不应该自动出现在画布上

# 已移除：
# - 收集需要自动排列的节点逻辑
# - start_x/start_y 自动排列计算
# - NodeItem 创建与添加
# - auto_arranged_count 计数
```

### 修复 5：增加详细诊断日志

```python
# ui/canvas/canvas_layout.py - load_layout finally 块中

logger.info(
    f"[load_layout] 画布节点诊断:\n"
    f"  总节点数: {len(self.nodes)}\n"
    f"  总连线数: {len(self.edges)}\n"
    f"  sceneRect: {self.scene.sceneRect()}"
)

# 输出每个节点的位置和可见性（最多前 5 个，避免日志过长）
for i, (name, node) in enumerate(list(self.nodes.items())[:5]):
    pos = node.pos()
    rect = node.rect() if hasattr(node, 'rect') else '(n/a)'
    visible = node.isVisible() if hasattr(node, 'isVisible') else True
    z = node.zValue() if hasattr(node, 'zValue') else 0
    logger.info(
        f"  [{i+1}] {name}: pos=({pos.x():.0f},{pos.y():.0f}) "
        f"visible={visible} z={z}"
    )
if len(self.nodes) > 5:
    logger.info(f"  ... 还有 {len(self.nodes) - 5} 个节点")
```

## 关键设计决策

### 决策 1：画布节点只来源于 canvas_layout.json
- **理由**：画布是用户的"可视化工作区"，只有用户手动拖入/移动的节点才应该出现在画布上
- **影响**：项目中所有节点在"节点列表面板"中显示；画布上只显示用户选择的节点
- **回滚点**：如果将来需要"全部节点自动显示在画布上"，只需在 `load_layout` 中重新加入从 `nodes_data` 创建 NodeItem 的逻辑

### 决策 2：用 try/finally 而非 try/except 包裹 setUpdatesEnabled
- **理由**：`setUpdatesEnabled(True)` 必须在**任何情况**下都被调用（包括异常）
- **影响**：无功能影响，只是代码健壮性增强
- **回滚点**：移除 try/finally 即回到旧行为

### 决策 3：在 finally 中同时刷新 scene 和 viewport
- **理由**：Qt 的渲染层有多层缓存（scene 的 BSP 树、viewport 的像素缓冲区）
- **影响**：可能造成轻微的额外渲染开销，但相比"画布为空"的严重问题，开销完全可以接受
- **回滚点**：如果性能测试发现过度刷新，可只保留 `scene.update(sceneRect())`
