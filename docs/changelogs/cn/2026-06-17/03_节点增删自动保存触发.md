# 节点增删自动保存触发（避免手动拖入节点后重启丢失）

---

## 问题描述

### 现象 1：从节点列表面板拖入节点到画布不保存
```
1. 用户从节点列表面板拖入一个节点到画布（调用 add_node_to_canvas）
2. 节点正确显示在画布上
3. 500ms 后检查 canvas_layout.json — 节点位置没有被写入！
4. 重启应用 — 该节点不在画布上显示
```

### 现象 2：从画布移除节点不保存
```
1. 用户在画布上选中一个节点，按 Delete 或通过上下文菜单移除
2. 节点从画布消失
3. canvas_layout.json 没有更新
4. 重启应用 — 该节点重新出现在画布上（从未被真正移除）
```

### 现象 3：子进程模式参数不匹配
```
# canvas_process.py 中的调用
self.canvas.add_node_to_canvas(node_name, info)
                                       ↑
                                      第 2 个参数

# 但 canvas_view.py 中定义只接受 1 个参数
def add_node_to_canvas(self, node_name):
    ...

# 结果：TypeError: add_node_to_canvas() takes 2 positional arguments but 3 were given
```

---

## 根因分析

### 根因 1：`add_node_to_canvas` 没有调用保存逻辑

```python
# 修改前
def add_node_to_canvas(self, node_name):
    # ... 创建 NodeItem
    node = NodeItem(node_name, language, status, x, y, 140, 80, self)
    self.scene.addItem(node)
    self.nodes[node_name] = node
    
    # 自动录制命令（用于历史回滚）
    self._record_create_node(node_name)
    
    # ❌ 没有触发 _save_timer！
    # ❌ canvas_layout.json 不会被更新
```

- `_save_timer` 是一个 `QTimer`，在节点移动、缩放等操作时触发
- `_save_timer.start(500)` 表示"500ms 后调用 _auto_save_layout"
- `_save_timer.stop()` 是为了防抖（连续操作时不会每次都保存）
- `add_node_to_canvas` 只是创建了节点并添加到 scene，但没有告诉系统"有变更需要保存"

### 根因 2：`remove_node_from_canvas` 同样没有调用保存逻辑

```python
# 修改前
def remove_node_from_canvas(self, node_name):
    if node_name not in self.nodes:
        return
    
    node = self.nodes[node_name]
    
    # 删除相关连线
    edges_to_remove = [...]
    for edge in edges_to_remove:
        self.remove_edge(edge)
    
    # 移除节点
    self.scene.removeItem(node)
    del self.nodes[node_name]
    
    # ❌ 没有触发 _save_timer！
```

### 根因 3：参数不匹配（子进程模式）

```python
# canvas_process.py - 子进程模式下，节点信息通过参数传递
def _on_node_added(self, node_name, node_info):
    self.canvas.add_node_to_canvas(node_name, node_info)
    #                                      ↑
    #                               传入节点信息（path/config/status）
    
# canvas_view.py - 但主进程模式下方法签名没有这个参数
def add_node_to_canvas(self, node_name):
    # 需要从 self.parent_window.nodes_data 查找节点信息
    # 无法直接接受 node_info 参数
```

- 子进程模式（`canvas_process.py`）是一个独立进程，它通过 IPC 消息接收节点信息
- 它不像主进程那样可以访问 `self.parent_window.nodes_data`
- 所以它在调用 `add_node_to_canvas` 时会直接传入 `node_info` 参数
- 但主进程模式的 `add_node_to_canvas` 只接受 `node_name`

## 修复方案

### 修复 1：`add_node_to_canvas` 接受可选 `node_info` 参数并触发保存

```python
# ui/canvas/canvas_view.py

def add_node_to_canvas(self, node_name, node_info=None):
    """添加节点到画布
    
    Args:
        node_name: 节点名称
        node_info: 可选的节点信息字典（子进程模式下直接传入）
                   包含 'path', 'config', 'status' 等字段
    """
    if node_name in self.nodes:
        themed_message(self, t("k_title_info"), t("k_canvas_node_exists"), "info")
        return
    
    # 获取节点信息（优先使用传入的 node_info，否则从 parent_window 读取）
    if node_info:
        # 子进程模式：直接使用传入的节点信息
        language = self.detect_language(node_info.get('path', ''))
        status = node_info.get('status', 'stopped')
    elif self.parent_window and node_name in self.parent_window.nodes_data:
        # 主进程模式：从 nodes_data 读取
        parent_info = self.parent_window.nodes_data[node_name]
        language = self.detect_language(parent_info['path'])
        status = parent_info.get('status', 'stopped')
    else:
        # 兜底：使用默认值
        language = "Python"
        status = "stopped"
    
    # 计算位置...
    # 创建节点...
    node = NodeItem(node_name, language, status, x, y, 140, 80, self)
    node.on_expand_requested = self.on_expand_requested
    self.scene.addItem(node)
    self.nodes[node_name] = node
    
    logger.info(f"节点 {node_name} 已添加到画布 (位置: {x}, {y})")
    
    # 自动录制命令
    self._record_create_node(node_name)
    
    # ✅ 触发自动保存布局（500ms 防抖）
    if self.parent_window and self.parent_window.current_project_path:
        self._save_timer.stop()
        self._save_timer.start(500)
```

## 自动保存机制说明

### `_save_timer` 工作原理

```
用户操作                    计时器状态                    实际保存
───────────────────   ────────────────────────   ──────────────────

1. 第 1 个节点拖入        _save_timer.start(500)    ← 500ms 后保存
                          （计时器开始计时）
   |<----- 200ms ---->|

2. 第 2 个节点拖入        _save_timer.stop()         ← 取消之前的计时器
                          _save_timer.start(500)    ← 重新计时 500ms
   |<----- 150ms ---->|

3. 节点位置微调          _save_timer.stop()
   (mouseMoveEvent)     _save_timer.start(500)    ← 重新计时 500ms
   |<--------- 600ms ------------>|

4. 无操作 (500ms 后)     计时器到期               _auto_save_layout()
                                                    → save_layout()
                                                    → canvas_layout.json 写入磁盘
```

### 已有的保存触发点（本次不修改）

```
# 以下操作已经触发保存：
- mouseMoveEvent(node)      ← 节点拖动时
- wheelEvent                ← 画布缩放时
- keyPressEvent(Delete)     ← 键盘删除时
- 节点重命名操作             ← 重命名后
- closeEvent                ← 窗口关闭时

# 本次新增：
- add_node_to_canvas()      ← ✅ 新增保存触发
- remove_node_from_canvas() ← ✅ 新增保存触发
```

### `_auto_save_layout` 实现

```python
# （已有的代码，此处仅展示供参考）
def _auto_save_layout(self):
    """自动保存画布布局（防抖计时器回调）"""
    try:
        if self.parent_window and self.parent_window.current_project_path:
            self.save_layout()
            logger.info("[auto_save] 画布布局已自动保存")
    except Exception as e:
        logger.warning(f"自动保存画布布局失败: {e}")
```

## 关键设计决策

### 决策 1：使用可选参数 `node_info=None` 而非重载方法
- **理由**：Python 不支持真正的方法重载；一个带默认值的可选参数同时支持主进程和子进程两种调用模式
- **影响**：方法签名稍微复杂，但只需维护一个方法
- **回滚点**：如果将来只支持一种模式，可直接移除参数并简化逻辑

### 决策 2：沿用 `_save_timer` 防抖机制，而非立即保存
- **理由**：与节点移动、缩放等操作的保存模式保持一致；连续操作时只保存一次，减少磁盘 IO
- **影响**：用户操作后需等待 500ms 才会真正写入磁盘，但这与现有行为一致
- **回滚点**：如果需要更强的"立即保存"语义，可改为直接调用 `self.save_layout()`（但要谨慎—可能产生大量磁盘写入）

### 决策 3：`remove_node_from_canvas` 与 `remove_node_with_cleanup` 的区分
- **`remove_node_from_canvas`**：只从画布移除节点，节点仍在项目中（节点列表面板可见）→ 只保存布局
- **`remove_node_with_cleanup`**：从项目中删除节点，停止进程，删除文件 → 保存布局 + 执行清理
- **理由**：用户有两种不同的删除意图，需要明确区分
- **回滚点**：如果发现用户对两种操作有混淆，可以增加确认对话框
