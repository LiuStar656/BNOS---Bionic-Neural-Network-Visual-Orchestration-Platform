# 架构解耦与功能统一化

## 📋 更新概述

本次更新对 BNNPOS 进行了系统性的架构解耦，引入 EventBus、DI 容器、ShutdownOrchestrator 等基础设施模块，消除了两大面板间 ~800 行重复代码，并将 150+ 菜单操作统一注册到 ActionRegistry 供画布菜单、节点菜单和菜单栏共用。

---

## 🎯 核心架构改进

### 1. EventBus 事件总线（Observer 模式）

**新增 `ui/core/event_bus.py`（45 行）**

- 发布-订阅模式的事件总线
- 支持 `subscribe` / `publish` / `unsubscribe`
- 通配符支持，解耦模块间直接依赖

### 2. DI/IOC 依赖注入容器

**新增 `ui/core/di.py`（80 行）**

- `DIContainer` — 轻量依赖注入容器
- `IConfig` 接口 — 配置读写抽象
- `JsonFileConfig` — JSON 文件配置实现
- `AppConfig` 单例注册为 `IConfig` 实现，DI 容器持有真实应用配置

### 3. PanelManager 面板管理器（Registry + Factory）

**新增 `ui/core/panel_manager.py`（131 行）**

- `PanelRegistry` — 统一管理面板注册
- `PanelManager` — 面板生命周期管理（创建、切换、保存状态）
- 注册 `node_list` / `resource_monitor` / `node_monitor` 三个面板
- 修复 `builtin_view_actions.py` 中的 `toggle_node_list_panel()` 调用

### 4. NodeControlService 节点控制服务（Bridge 模式）

**新增 `ui/core/node_control_service.py`（142 行）**

- `NodeStatus` 枚举 — running / idle / stopped / error
- `start_node()` / `stop_node()` / `stop_all_nodes()`
- 状态订阅机制，通过桥接模式与现有启停流程同步

### 5. ShutdownOrchestrator 关闭编排器

**新增 `ui/core/shutdown_orchestrator.py`（35 行）**

- 声明式关闭步骤编排
- 注入 7 个真实关闭步骤替代 `closeEvent` 中内联代码
- `_restart_application` 复用同一组关闭方法，消除 ~70 行重复

### 6. Canvas 控制器组合（Composition over Inheritance）

**新增 `ui/canvas/controllers.py`（115 行）**

- 7 个控制器通过委托模式替代 Mixin 继承：
  - `CanvasConnectionController` — 连线管理
  - `CanvasBatchOperations` — 批量操作
  - `BoxSelectionController` — 框选
  - `CanvasMenuController` — 右键菜单
  - `CanvasLayoutController` — 布局保存/恢复
  - `CanvasColorController` — 颜色设置
  - `CanvasZoomController` — 缩放

### 7. NodeListOperationsMixin 共享 Mixin

**新增 `ui/panels/node_list_ops.py`（609 行）**

- 提取 `node_list_dock.py` 和 `node_list_panel.py` 中 ~800 行重复代码
- 33 个共享方法：节点树操作、删除、组操作、批量操作
- `node_list_dock.py`：~960 → 264 行
- `node_list_panel.py`：~1,197 → 433 行
- 净减少 ~364 行（扣除新增基础设施）

## 🌟 功能统一化

### 8. Action 系统全面扩展

**扩展 `ui/core/actions/builtin_node_actions.py`（6 → 30 个 Action）**

新增 24 个节点/组操作定义：
- 节点操作：`add_to_canvas` / `open_folder` / `view_log` / `edit_config` / `rename` / `delete` / `unmount`
- 批量操作：`batch_add_to_canvas` / `batch_open_folders` / `batch_view_logs` / `batch_edit_configs` / `batch_delete`
- 选择操作：`select_all` / `deselect_all`
- 组操作：`create` / `move_to` / `batch_move_to` / `remove_from` / `batch_remove_from` / `rename` / `delete` / `start` / `stop` / `toggle_expand`
- 未分组操作：`start` / `stop`

**扩展 `ui/core/actions/builtin_canvas_actions.py`（7 → 20 个 Action）**

新增/修复 13 个画布操作：
- 连线操作：`delete_edge` / `change_edge_color`（原为空壳，已修复） / `start_connection`
- 画布操作：`remove_node` / `batch_remove` / `expand_node` / `toggle_draw_toolbar` / `monitor`
- 新建节点：`new_node.{Python,Rust,Node.js,Go,Java,C++,Shell}` 7 个
- 画布颜色：`bg_color` / `grid_color` / `default_edge_color` 3 个

### 9. 右键菜单全面统一

**重构 `ui/panels/node_list_context.py`**

- 静态名称项全部走 `ActionFactory.create_action()`
- 动态名称项（含 `{count}`）通过 `_dispatch()` 路由到 `ActionRegistry.execute()`
- 三类菜单（单节点 / 批量 / 组）均从 ActionRegistry 生成

**重构 `ui/canvas/canvas_menus.py`**

- 4 种菜单场景统一走 Action 系统
- 仅样式 / 节点颜色 / 终端子菜单保持手动 QAction（需 `NodeItem` 引用或平台判断）
- 消除 `delete_edge` / `change_edge_color` 空壳 Action

### 10. 画布菜单与节点列表菜单重复消除

两个右键菜单现在共用同一套 Action 注册表：

```
菜单栏 ──┐
快捷键 ──┼──► ActionRegistry ──► execute_fn ──► panel / canvas / main_window
画布菜单 ─┤      (50 Actions)
节点列表 ─┘
```

## ⚠️ 注意事项

- `AppConfig` 通过鸭式类型实现 `IConfig`，两个文件的 `get/set/save` API 兼容
- `node_list_ops.py` 中 33 个方法通过 Mixin 注入两个面板，不改变外部调用
- 画布样式和节点颜色子菜单因需要 `NodeItem` 引用，保持手动 QAction
- 终端子菜单因平台差异，保持手动构建
