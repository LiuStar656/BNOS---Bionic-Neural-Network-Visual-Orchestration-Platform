# 🔧 强制删除节点文件夹

## 🔧 强制删除节点文件夹 (2026-06-05)

### 问题描述

**节点文件夹被占用导致删除失败**
- **问题**：删除节点时，由于文件夹被未知程序占用（如文件资源管理器、杀毒软件、Python 进程等），导致删除失败
- **影响**：用户无法正常删除节点，影响使用体验

### 修复方案

**三层强制删除机制**

1. **第一层：尝试重命名**
   - 将文件夹重命名为临时名称，绕过某些占用场景

2. **第二层：Windows rmdir 命令**
   - 使用 `rmdir /s /q` 命令强制删除
   - Windows 专用，可处理文件被占用的情况

3. **第三层：扫描并终止占用进程**
   - 使用 psutil 扫描所有进程
   - 检查进程的工作目录和打开的文件是否包含节点路径
   - 强制终止这些进程后重试删除

def _force_stop_node_processes(self, node_path):
    """强制停止可能占用节点文件夹的进程"""
    import psutil
    
    for proc in psutil.process_iter(['pid', 'name', 'open_files', 'cwd']):
        try:
            # 检查工作目录
            cwd = proc.cwd()
            if node_path_lower in cwd.lower():
                proc.kill()
            
            # 检查打开的文件
            for f in proc.open_files():
                if node_path_lower in f.path.lower():
                    proc.kill()
                    break
        except (psutil.AccessDenied, psutil.NoSuchProcess):
            pass
