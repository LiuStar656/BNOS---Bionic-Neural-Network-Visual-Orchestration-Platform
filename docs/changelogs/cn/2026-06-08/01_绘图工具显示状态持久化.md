# 🎨 绘图工具显示状态持久化功能

## 📋 问题概述

之前绘图工具栏的显示状态没有持久化保存，每次重启软件都需要重新按 `D` 键显示工具栏，用户体验不佳。此外还存在需要连续两次操作才能切换状态的问题。

## ✨ 功能实现

### 1. **配置文件集成**

在 `app_config.json` 中新增 `draw_toolbar_visible` 字段，默认值为 `false`。

### 2. **状态保存机制**

修改 `ui/canvas/draw_layer.py`：
- `show_toolbar()` 方法在显示工具栏时自动保存配置
- `hide_toolbar()` 方法在隐藏工具栏时自动保存配置
- 新增 `_save_toolbar_config()` 私有方法处理配置持久化

### 3. **状态恢复机制**

修改 `ui/canvas/canvas_view.py`：
- 新增 `_load_draw_toolbar_config()` 方法，从配置读取初始状态
- 在初始化 `draw_layer` 后立即调用恢复方法

### 4. **修复两次操作问题**

优化了 `show_toolbar()` 和 `hide_toolbar()` 的逻辑：
- 移除了冗余的条件判断
- 确保一次操作就能生效
- 避免了内部状态与实际显示不同步的问题

## 🔧 技术细节

### 修改的文件

| 文件 | 说明 |
|------|------|
| `ui/core/app_config.py` | 添加 `draw_toolbar_visible` 配置项 |
| `ui/canvas/draw_layer.py` | 工具栏状态管理和保存 |
| `ui/canvas/canvas_view.py` | 状态恢复加载 |
| `ui/canvas/canvas_layout.py` | 移除冲突的状态加载逻辑 |

### 配置字段

```json
{
  "draw_toolbar_visible": false
}
```

## 🎯 功能特性

✅ **持久化存储** - 状态自动保存到 `app_config.json`
✅ **首次启动** - 软件启动时自动读取配置
✅ **实时更新** - 用户切换显示/隐藏时自动保存
✅ **一次操作** - 修复了需要连续两次操作的问题
✅ **重启保持** - 重启软件后保持上次的显示状态
✅ **配置管理** - 与其他全局配置统一管理

## 📝 用户指南

### 使用方法

1. **显示/隐藏绘图工具栏**
   - 快捷键：按 `D` 键
   - 或通过画布右键菜单选择

2. **状态自动保存**
   - 每次切换后状态会自动保存
   - 无需手动操作

3. **重启恢复**
   - 下次启动时会自动恢复上次的显示状态
