# P2 级别优化：主窗口拆分

## 概述

将主窗口文件（1519行）拆分为三个职责单一的模块，使用 Python Mixin 模式实现职责分离。

## 优化内容

### 新建文件

**`ui/main_window_state.py`** - 状态管理模块

负责窗口状态的保存和恢复：
- `save_window_state()` - 保存窗口状态
- `restore_window_state()` - 恢复窗口状态
- `_restore_panel_state()` - 恢复面板状态（立即创建 Dock）
- `_save_panel_visibility()` - 保存面板可见性
- `auto_open_last_project()` - 自动打开上次项目

**`ui/main_window_lifecycle.py`** - 生命周期管理模块

负责窗口的启动和关闭流程编排：
- `_init_and_restore()` - 初始化和恢复流程
- `closeEvent()` - 关闭事件处理
- `moveEvent()` - 移动事件处理
- `resizeEvent()` - 大小改变事件处理
- `_force_stop_all_nodes()` - 强制停止节点

**`ui/main_window_actions.py`** - 业务操作模块

负责各种业务操作：
- `_create_node_list_panel()` - 创建节点列表面板
- `_create_resource_monitor()` - 创建资源监测面板
- `_create_node_monitor()` - 创建节点监测面板
- `_show_message_box()` - 显示消息对话框
- `_handle_node_created()` - 处理节点创建事件
- `_handle_node_deleted()` - 处理节点删除事件
- `_handle_node_status_changed()` - 处理节点状态变更

### 修改文件

**`ui/main_window.py`**

- 引入三个 Mixin 类：`MainWindowStateMixin`、`MainWindowLifecycleMixin`、`MainWindowActionsMixin`
- 主窗口类继承这三个 Mixin，实现职责分离

## 设计特点

1. **职责分离**：每个模块职责单一，便于维护和测试
2. **Mixin 模式**：使用 Python 多重继承实现行为复用
3. **渐进式拆分**：保留原有接口，不影响现有调用
4. **向后兼容**：所有方法保持原有签名
