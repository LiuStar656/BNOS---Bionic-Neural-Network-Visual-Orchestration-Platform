# 项目打开异步化与画布布局加载修复

## 概述

将 `project_open()` 从**同步阻塞调用**改为**异步定时器驱动**的加载流程，解决用户反馈的"同时打开两个项目后，第二个项目的画布内布局未能正常加载"的问题。

---

## 1. 项目打开异步化

### 变更前的行为

```python
# project_manager.py - 旧版本
def project_open(main_window, project_path=None):
    if not project_path:
        project_path = pick_folder(main_window, ...)  # 模态对话框阻塞主线程
    project_refresh(main_window, async_mode=False)    # 同步扫描所有节点
    project_load_layout(main_window)                  # 同步加载布局
    show_toast(...)
```

**问题**：
- `project_refresh(async_mode=False)` 在主线程同步扫描所有节点的进程状态，节点数量多时 UI 卡顿
- `load_layout` 在画布的 QGraphicsScene 尚未完全初始化时执行，节点布局为空

### 变更后的行为

```python
# project_manager.py - 新版本
def project_open(main_window, project_path=None):
    if not project_path:
        project_path = pick_folder(main_window, ...)

    # 立即提示
    show_toast(main_window, "正在打开项目...", "info")

    # 用 QTimer.singleShot 切回事件循环，等 GUI 初始化完成后再加载数据
    QTimer.singleShot(10, lambda: _open_project_async(main_window, project_path))

def _open_project_async(main_window, project_path):
    project_refresh(main_window, async_mode=True)     # 异步扫描（与节点启停一致）
    # 等 Dock 面板 ready 后再恢复布局
    QTimer.singleShot(300, lambda: _after_open(main_window))
```

**核心改进**：
- `async_mode=True`：节点状态检测通过 `QTimer` 轮询实现，不阻塞主线程
- `QTimer.singleShot(10ms + 300ms)`：分阶段执行，确保 GUI 事件循环推进、Dock 面板就绪
- 主窗口不再"冻结"，可看到 Toast 通知与节点列表逐步刷新

## 2. 画布布局加载修复（第二个项目的根因）

### 根因分析

第二个项目打开后画布内节点为空，根因在 `canvas_host.py` 的信号触发顺序：

```python
# canvas_host.py - 旧代码（bug）
def _create_canvas_dock(self, project_path, project_name):
    # ... 创建画布 ...
    canvas_changed.emit()                                    # ← 触发 sync_canvas_data_to_main_window
                                                             #    将 nodes_data = {} 回写主窗口！
    canvas.load_layout(layout_path)                          # ← 晚一步加载节点，已经被清空了
    update_canvas_data_from_main_window()                    # ← 从主窗口拷回空数据
```

**执行时序**：
1. `canvas_changed.emit()` → `sync_canvas_data_to_main_window()` → 主窗口的 `nodes_data[project_path] = {}`（空 dict）
2. `canvas.load_layout()` → 正确加载节点到 canvas 内部，但没人再告诉主窗口
3. `update_canvas_data_from_main_window()` → 把步骤 1 的空 dict 回写到 canvas

### 修复：调换信号触发顺序

```python
# canvas_host.py - 修复后
def _create_canvas_dock(self, project_path, project_name):
    # ... 创建画布 ...
    canvas.load_layout(layout_path)                          # 先加载节点布局 ✅
    update_canvas_data_from_main_window()                    # 再同步数据（nodes_data 已有内容）
    canvas_changed.emit()                                    # 最后广播变更（通知面板刷新）
```

**同时在 project_manager.py 中，确保 `_create_canvas_dock` 在 `project_refresh(async_mode=True)` 之后调用**，先有节点数据，再恢复布局。
