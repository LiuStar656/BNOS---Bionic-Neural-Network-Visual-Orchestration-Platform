# 节点样式统一化、锚点坐标修复与进程生命周期保护

## 概述

本日更新聚焦三个方向：
1. **节点样式系统精简**：彻底删除矩形/圆点样式，全系统统一为面板模式（DetailedNodeStyle）
2. **锚点坐标修正**：锚点位置改为节点两侧边线中点，并修复 setPos/_find_nearest 双倍偏移 bug
3. **进程生命周期保护**：修复 TerminalProcess 析构时 QProcess 已销毁导致的 RuntimeError

---

## 1. 节点样式系统精简

### 变更动机

原有三种节点样式（面板/矩形/圆点）带来维护复杂度和不一致的用户体验。面板模式已集成了所有核心能力（参数直显、状态灯、CPU/MEM 占用、语言标签），保留其他样式无实际价值。

### 变更内容

| 操作 | 文件 | 说明 |
|------|------|------|
| **删除** | `ui/canvas/items/styles/rect.py` | RectNodeStyle、DarkRectNodeStyle、LightRectNodeStyle |
| **删除** | `ui/canvas/items/styles/dot.py` | DotNodeStyle |
| **重写** | `ui/canvas/items/styles/detailed.py` | 改为直接继承 `NodeStyle` 基类（原继承 RectNodeStyle） |
| **简化** | `ui/canvas/items/styles/__init__.py` | StyleRegistry 仅注册 `"detailed"` → `DetailedNodeStyle` |
| **重写** | `ui/canvas/items/node_item.py` | `set_style` 简化为仅用 DetailedNodeStyle；`paint`/`shape`/`mousePressEvent` 移除 is_dot 判断；构造函数默认使用 DetailedNodeStyle |
| **更新** | `ui/canvas/canvas_layout.py` | 默认样式 `"rect"` → `"detailed"` |
| **简化** | `ui/canvas/items/node_style.py` | 兼容层只导出 DetailedNodeStyle + StyleRegistry |

## 2. 锚点坐标修正：两侧边线中点

### 变更动机

输入/输出锚点原先的回落位置分别为左上角和右下角，不统一且难以定位。统一改为节点左边线中点（x=0, y=h/2）和右边线中点（x=nw, y=h/2）。

### 坐标系统修复（根因 Bug）

**问题**：`_make_anchor()` 创建的 `AnchorItem(-size/2, -size/2, size, size)` 本地椭圆中心在 `(0,0)`，因此 `setPos(px, py)` 后视觉中心就是 `(px, py)`。但旧代码在所有 `setPos` 调用中多减了一个 `size/2`：

```python
# 旧代码（有 bug）
anchor.setPos(center_x - size/2, center_y - size/2)  # 视觉中心偏移了 size/2
```

同时 `_find_nearest` 又加了 `half` 补偿回来：

```python
# 旧代码（补偿 bug）
center_x = anchor.pos().x() + half  # 补偿上面多减的 size/2
```

两个 bug 互相抵消，命中检测正常工作，但视觉位置始终偏移 8px。

**修复**：统一移除 `setPos` 中的 `-size/2` 和 `_find_nearest` 中的 `+half`，让坐标值即所见。

### 变更文件

| 文件 | 修改点 |
|------|--------|
| `ui/canvas/items/anchor_manager.py:228` | 主输入锚点回落位置：`(ANCHOR_HALF+8, ANCHOR_HALF+4)` → `(0, nh/2)` |
| `ui/canvas/items/anchor_manager.py:288` | 主输出锚点回落位置：`(nw, nh - ANCHOR_HALF)` → `(nw, nh/2)` |
| `ui/canvas/items/anchor_manager.py:228,254,288` | 三处 `setPos(cx - size/2, cy - size/2)` → `setPos(cx, cy)` |
| `ui/canvas/items/anchor_manager.py:643` | `_find_nearest` 移除 `+half` 补偿 |
| `ui/canvas/items/node_item.py:687-689` | `__output__` 行锚点：`(final_w - ANCHOR_SIZE/2, final_h - ANCHOR_HALF)` → `(final_w, final_h/2)` |

## 3. 进程生命周期保护

### 问题

`TerminalProcess.__del__()` → `stop()` → `self.process.state()` 在程序退出时可能访问已被 Qt 父控件销毁链提前释放的 `QProcess` C++ 对象，抛出：

```
RuntimeError: libshiboken: Internal C++ object (PySide6.QtCore.QProcess) already deleted.
```

### 修复

在 `terminal_process.py` 的 `stop()` 和 `__del__()` 中添加 `try/except RuntimeError` 保护，C++ 对象已销毁时静默返回。
