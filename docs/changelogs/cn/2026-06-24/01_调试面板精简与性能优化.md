# 调试面板精简与性能优化

## 一、调试面板移除

### 背景

调试面板经过多轮迭代（节点选择器、日志断点、会话管理、实时日志管道接入），发现其核心功能（日志查看、节点启停）与已有面板高度重叠：
- 节点启停 → 节点列表面板、属性面板已有完整功能
- 日志查看 → `NodeMonitorDock` 中的 `NodeLogSubPanel` 功能更完整（含资源占用监控）
- 会话表格、变量表格 → 无数据源，始终为空

仅有一个独立功能：**日志关键字高亮匹配**，不足以支撑一个完整面板。

## 二、性能分析面板异步化

### 问题

打开性能分析面板时整个软件明显卡顿。根因是 `psutil.process_iter()` 在主线程调用，Windows 上首次枚举所有进程需要 1-3 秒，直接冻结 UI。

### 修复

`PerformancePanel` 中的 `StatsCollectorThread` 新增 `processes_ready` 信号：
- 进程列表采集从主线程 `QTimer` 移入后台线程
- 面板仅通过 `processes_ready.connect()` 接收数据后更新表格
- `psutil` 调用完全移至后台线程

### 修改文件

- `ui/panels/performance_panel.py`

## 三、面板生命周期防悬空指针

### 问题

面板关闭后 Qt 销毁 C++ 对象，但 `self.xxx_panel` Python 引用仍指向已死对象。原 `hasattr + is None` 检查无法检测 dangling pointer，导致 `Internal C++ object already deleted` 崩溃。

### 修复

`ui/main_window/panel.py` 添加 `_is_panel_alive()` 静态方法，使用 `shiboken6.isValid()` 检测 C++ 对象是否存活。所有 `show_*` 方法和 `toggle_node_list_panel` 统一改用：

```python
if not self._is_panel_alive(getattr(self, 'xxx_panel', None)):
    # 重建面板
```

同时 `_on_dock_panel_closed` 中所有面板属性访问改为 `getattr(self, 'xxx', None)` 防止从未打开的面板触发 `AttributeError`。

### 修改文件

- `ui/main_window/panel.py`

## 四、菜单面板打钩标记修复

### 问题

重启后自启面板的菜单项只有"节点列表"正确显示打钩，性能分析、调试、预设库面板的菜单项没有打钩标记。

### 根因

`is_checked_fn` 在菜单构建时只评估一次，此时面板尚未创建。`node_list` 能正常工作是因为额外带了 config fallback。

### 修复

所有 6 个 Dock 面板菜单项统一采用 `运行时可见性 || config 持久化状态` 双重检查：

| 面板 | is_checked_fn |
|------|--------------|
| `performance_panel` | `config.performance_dock` |
| `template_selector` | `config.preset_library_dock` |
| `node_monitor_dock` | runtime visible + config fallback |
| `resource_monitor` | runtime visible + config fallback |
| `node_list` | 已正常工作 |

### 修改文件

- `ui/core/actions/builtin_view_actions.py`
