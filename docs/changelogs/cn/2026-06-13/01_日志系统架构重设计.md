# 日志系统架构重设计

## 问题背景

原日志系统存在以下痛点：

1. **单文件无限膨胀**：`bnos_console.log` 随时间增长至 4.78MB+，无轮转机制
2. **错误日志混杂**：ERROR 与 INFO 混在同一文件，排查问题需从海量日志中翻找
3. **编码错误**：emoji 字符在 Windows GBK 终端触发 `UnicodeEncodeError`
4. **高频噪音**：鼠标交互每秒数十条 DEBUG，71% 为无意义日志
5. **无自动清理**：旧日志永久堆积

核心矛盾：既要保留完整日志用于排查，又不能让文件无限膨胀。

---

## 新架构

### 文件分离

```
logs/
├── bnos.log              ← 日常日志 (INFO+, 按天轮转, 保留7天)
├── bnos.log.YYYY-MM-DD   ← 历史日常日志 (自动归档)
└── bnos_error.log        ← 错误日志 (ERROR+, 按大小轮转, 1MB×5备份)
```

### 三层防臃肿机制

```
调用 logger.xxx()
     │
     ▼
┌─────────────────────┐
│ ① FrequencyFilter   │  相同消息 30s 内 >5 次 → 抑制
│    ERROR/CRITICAL   │  永不过滤，确保错误不丢失
│    永远放行          │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ ② DebugLevelManager │  高频画布模块默认静默 DEBUG
│    可运行时切换      │  set_debug_mode(True) 全开
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ ③ 轮转/清理          │  bnos.log 按天 → 旧文件自动删除
│    bnos_error.log    │  按 1MB → 旧备份自动截断
│    按大小            │
└─────────────────────┘
```

## 核心组件

### SafeStreamHandler

```python
class SafeStreamHandler(logging.StreamHandler):
    """安全控制台处理器，自动处理 Windows GBK 编码下的特殊字符"""
```

Windows 控制台使用 GBK 编码，使用 `errors='replace'` 替换无法编码的字符，兜底防御编码错误。

### FrequencyFilter

```python
class FrequencyFilter(logging.Filter):
    """相同日志在 30s 窗口内超过 5 次后抑制，ERROR/CRITICAL 永远放行"""
```

- 日志键：`(级别, 消息摘要, 文件名)`
- 前 5 次正常输出
- 第 6 次输出一条汇总 WARNING：`[FREQ] 日志过于频繁...后续抑制`
- 第 7 次起丢弃
- **ERROR/CRITICAL 永不过滤** — 保证错误不丢失

### DebugLevelManager

```python
class DebugLevelManager(logging.Filter):
    """模块级 DEBUG 控制器"""
```

- 默认静默模块：`canvas_view`, `canvas_layout`, `edge_item`, `node_item`, `anchor_item`, `polling_manager`
- `set_debug_mode(True)` 运行时全开
- `add_quiet_module()` / `remove_quiet_module()` 动态增删

### _cleanup_old_logs

启动时扫描 `logs/bnos.log.*` 文件，删除 7 天前的归档文件。

## PollingManager super().__init__() 修复

**错误**：`RuntimeError: super-class __init__() of type PollingManager was never called`

**原因**：单例检查 `if PollingManager._initialized: return` 在 `super().__init__(parent)` 之前，导致 Qt C++ 层未正确初始化。

**修复**：`polling_manager.py` 中 `super().__init__(parent)` 移到单例检查之前。

```python
def __init__(self, parent=None):
    super().__init__(parent)  # 必须在单例检查之前
    if PollingManager._initialized:
        return
    PollingManager._initialized = True
```

## 效果对比

| 指标 | 旧架构 | 新架构 |
|------|--------|--------|
| 单文件最大体积 | 无上限（实测 4.78MB） | < 1MB（双重约束） |
| 错误定位 | 需 grep 全量日志 | 直接查看 `bnos_error.log` |
| 旧日志清理 | 手动删除 | 启动自动清理 7 天前 |
| 高频噪音 | 无过滤 | FrequencyFilter 抑制 80%+ |
| 编码兼容 | 存在 UnicodeEncodeError | 双重保障 |
