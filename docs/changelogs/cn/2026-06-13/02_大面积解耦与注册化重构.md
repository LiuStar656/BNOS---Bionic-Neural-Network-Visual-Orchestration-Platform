# 大面积解耦与注册化重构

## 背景

经过数月积累，项目中多个模块形成了高度耦合的单体文件：

| 文件 | 行数 | 问题 |
|------|------|------|
| `builtin_node_actions.py` | 613 | 23 个 action 混在一个函数 |
| `parameter_widgets.py` | 570 | 10 种控件类 + 工厂 + 常量全在一个文件 |
| `node_style.py` | 523 | 3 种样式混在一个文件 |
| `graphic_items.py` | 298 | 6 种图形类 + 基类 + 常量共用一个文件 |

这些问题在修 bug 时尤为突出：改一个样式要翻 500 行文件，新增一个参数控件要在 570 行中找位置。

---

## 解决方案：统一 "分离+注册" 模式

所有重构遵循同一模式：**每种类型独立文件 → package `__init__.py` 注册表 → 兼容重定向旧文件**。

---

## 一、节点样式解耦

`node_style.py`(523行) → `items/styles/` 包(5文件)

```
ui/canvas/items/styles/
├── __init__.py        # StyleRegistry + re-export
├── _base.py           # NodeStyle 基类
├── rect.py            # RectNodeStyle + Dark/Light 变体
├── dot.py             # DotNodeStyle
└── detailed.py        # DetailedNodeStyle
```

**核心改动**: `isinstance(style, DotNodeStyle)` → `self._style.style_key == "dot"` 字符串比较，解耦对具体类的 import 依赖。

**StyleRegistry API**:
```python
StyleRegistry.get("dot")       → DotNodeStyle 类
StyleRegistry.get("unknown")   → DarkRectNodeStyle（fallback）
StyleRegistry.keys()           → ["rect", "dot", "detailed"]
```

新增样式仅需两步：新建 `newstyle.py` + 注册表加一行。

---

## 二、参数控件解耦

`parameter_widgets.py`(570行) → `parameter_widgets/` 包(14文件)

```
ui/canvas/parameter_widgets/
├── __init__.py         # WidgetRegistry + re-export
├── _base.py            # ParameterWidget 基类 + 常量
├── _proxy_combo.py     # QGraphicsProxyWidget 兼容 ComboBox
├── string.py           # StringWidget
├── text.py             # TextWidget（多行）
├── password.py         # PasswordWidget
├── int_widget.py       # IntWidget
├── float_widget.py     # FloatWidget
├── bool_widget.py      # BoolWidget
├── enum_widget.py      # EnumWidget
├── file_picker.py      # FilePickerWidget
├── dir_picker.py       # DirPickerWidget
├── color_widget.py     # ColorWidget
└── range_widget.py     # RangeWidget（滑块）
```

**WidgetRegistry API**:
```python
WidgetRegistry.get("int")      → IntWidget 类
WidgetRegistry.get("unknown")  → StringWidget（fallback）
WidgetRegistry.keys()          → 11 种参数类型
```

Consumer 侧零改动 — `from ui.canvas.parameter_widgets import ParameterWidget` 删旧 `.py` 后自动解析到包 `__init__.py`。

## 三、绘图图形解耦

`graphic_items.py`(298行) → `graphic_items/` 包(7文件)

```
ui/canvas/graphic_items/
├── __init__.py      # GraphicRegistry + re-export
├── _base.py         # GraphicBase + 常量(C_STROKE/C_FILL/C_TEXT/STROKE_W)
├── rect.py          # RectGraphic
├── round_rect.py    # RoundRectGraphic
├── polygon.py       # PolygonGraphic
├── arrow.py         # ArrowGraphic
└── text.py          # TextGraphic（直接继承 QGraphicsItem）
```

**GraphicRegistry API**:
```python
GraphicRegistry.get("arrow")       → ArrowGraphic 类
GraphicRegistry.from_dict(d)       → 统一反序列化入口
```

`GraphicBase.from_dict()` 保留，内部委托给 `GraphicRegistry.from_dict()`。

## 四、内置 Action 解耦

`builtin_node_actions.py`(613行) → `actions/node/` 子包(9文件)

```
ui/core/actions/node/
├── __init__.py        # register_node_actions() 汇总入口
├── _lifecycle.py      # start/stop/config/refresh/mount/export（6 action）
├── _context_menu.py   # add_to_canvas/open_folder/view_log/edit_config/rename/delete/unmount（7 action）
├── _batch.py          # 5 个 batch_* action
├── _selection.py      # select_all/deselect_all（2 action）
├── _group.py          # 10 个 group.* action
├── _ungrouped.py      # ungrouped.*（2 action）
├── _ide.py            # open_vscode/open_trae_ide（2 action）
└── _style.py          # change_style/4 color actions（4 action）
```

每个子模块导出 `register(main_window)` 函数，`__init__.py` 循环调用汇总。新增 action 只需在对应的子文件里加内部 `register()` 调用。

兼容性：旧文件改为 `from ui.core.actions.node import register_node_actions` 重定向，3 个 consumer 零改动。

---

## 五、面板共享模块提取

消除 Node/Robot Log SubPanel 和 Resource 采集的重复代码：

```
ui/panels/_shared/
├── node_log_sub_panel.py           # BaseNodeLogSubPanel 基类
├── node_panel_sync_mixin.py        # SyncMixin（_sync_panels 等）
└── system_resource_collector.py    # 共享资源采集器单例
```

- `NodeMonitor` 和 `NodeMonitorDock` 各自消除约 80-110 行重复代码
- `ResourceMonitor` 和 `ResourceMonitorDock` 消除 212 + 114 行
- `shared_resource_collector` 单例解决两个面板各自维护 `psutil` 状态导致的累积不一致

## 六、Bug 修复（同期）

| Bug | 根因 | 修复 |
|-----|------|------|
| 退出崩溃 `PollingManager` 无 `stop_all` | `application_context.py` 方法名拼错 | `stop_all`→`stop`, 构造函数→`instance()` |
| 退出崩溃 `ProcessManager` 无 `terminate_all` | 同上 | `terminate_all`→`stop_all` |
| 面板模式圆形残留 | `DotNodeStyle` 创建 `_body` 圆形，`DetailedNodeStyle.apply()` 漏了隐藏它 | 加 `_body.setVisible(False)` |
| 状态显示条不该在节点/面板模式出现 | `update_status()` 无条件重建 widget | 加 `status_show` 守卫 |
| 小锚点连线切换样式后跑到大锚点 | edge 迁移只用旧锚点 `port_name` 匹配 | 优先用 `_desired_target_port_name` 匹配 |
| `config.json` param/ports 被 `start.json` 覆盖丢失 | `start.json` 不含元数据字段，整体替换 config | 从磁盘重读 `config.json`，运行时字段覆盖 |
