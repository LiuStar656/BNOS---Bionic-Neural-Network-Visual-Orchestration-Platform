# 单模式自适应节点渲染 — 开发方案

> 版本：v1.0
> 日期：2026-06-15
> 范围：ui/canvas/items + ui/core/connection_inferrer + 相关样式切换菜单
> 原则：**config.json 驱动一切 → 节点（锚点 + 参数控件的有/无全部由 config 决定

---

## 1. 目标与原则

### 1.1 核心目标

将当前"三种可切换样式（方框/圆点/面板）"重构为"**一种自适应渲染**"：

- **不再有"用户可切换的"节点样式"**
- **节点在画布上的外观，完全由该节点的 `config.json` 内容决定
- 唯一保留两种"简化方框"紧凑小方框 → 详细面板"两种形态**，但对用户是同一种模式（自动选择
- **锚点的有/无、位置、数量 → 由 `listen_upper_file` / `output_file` 是否非空决定
- **参数控件的有/无 → 由 `parameters` / `input_ports`（多端口）是否存在决定

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **单一真实来源 | config.json 同时表达运行时行为与 UI 渲染 |
| **零配置默认 | 用户不需要选样式 |
| **最少改动 | 不触动非相关模块 |
| **向前兼容 | 老节点 config.json 仍能正常渲染 |
| **可调试 | 保留一个"强制展开/收起"调试开关 |

## 2. 形态判定规则

### 2.1 节点形态选择

```python
def detect_shape(config: dict) -> str:
    """
    返回：'simple' 或 'detailed'
    """
    # 条件 1：有 parameters → detailed
    if config.get("parameters"):
        return "detailed"

    # 条件 2：有 2 个及以上 node-source 输入端口 → detailed
    input_ports = config.get("input_ports", [])
    node_inputs = [p for p in input_ports if p.get("source") == "node"]
    if len(node_inputs) >= 2:
        return "detailed"

    # 其他：简化方框
    return "simple"
```

### 2.2 锚点判定

```python
def detect_anchors(config: dict) -> dict:
    """
    返回：{"input": bool, "output": bool}
    """
    has_input = bool(str(config.get("listen_upper_file", "")).strip())
    has_output = bool(str(config.get("output_file", "")).strip())
    return {"input": has_input, "output": has_output}
```

### 2.3 config.json 语义表

| 字段 | 是否影响渲染 | 影响 |
|-------|-------------|------|
| `node_name` | 是 | 节点名称 |
| `listen_upper_file` | 是 | 左侧输入锚点的有/无 |
| `output_file` | 是 | 右侧输出锚点的有/无 |
| `input_ports` | 是 | 详细模式下多行输入控件 + 多锚点 |
| `parameters` | 是 | 详细模式下参数行 |
| `filter` / `output_type` | 否 | 运行时字段，不影响 UI |
| `custom_*_color` | 是 | 自定义颜色（保留） |

## 3. 最终渲染规范

### 3.1 简化方框（simple）

```
┌──────────────────┐
│ ●  node_1         │  ← 左侧输入锚点（●）/输出锚点（●）
│     Python        │  ← 名称 + 语言/状态灯（●）
└──────────────────┘
      ↑ 120x56

特征：
- 尺寸：120px × 56px（比现在 140×120 更紧凑）
- 圆角：8px 圆角
- 颜色：深色背景 #2d2d30，边框 #454545
- 组件：节点名称 + 状态指示灯 + 语言文本 + 两个锚点
- 锚点：直径 12px，贴在节点左右边界垂直居中
- 状态灯：直径 8px，右上角
```

### 3.2 详细面板（detailed）

```
┌─────────────────────────────────────────────┐
│  node_python_1                    │  ← 标题栏 28px，按语言变色
├─────────────────────────────────────────────┤
│ ● input_data   [ ______________________ ] │  ← input_port 行 + 小锚点
│ ● input_cfg    [ ______________________ ] │
│   timeout      [ 30          ] │  ← parameter 行
│   enabled      [☑] │
│                              output ● │  ← 输出锚点在底部
└─────────────────────────────────────────────┘

特征：
- 宽度由内容自适应（最小 340px，由参数文本自然宽度）
- 标题栏高亮（语言色），底部留白 8px
- 左侧锚点区 16px + 边距 12px = 28px 内边距
- 每行高度统一 32px，行间距 2px
- 输入端口行：锚点直径 10px，贴着锚点
- 输出行在最底一行，右侧显示 output 文字 + 右侧大锚点 16px
```

def auto_detect(cls, config: dict) -> NodeStyle:
    """
    根据 config.json 内容自动选择节点样式。

    规则：
    - 有 parameters → detailed
    - 有 >=2 个 node-source input_port → detailed
    - 其他 → simple

    shape = detect_shape(config or {})

## 5. 兼容与迁移策略

### 5.1 老节点兼容

- config.json 只有 `node_name` + `listen_upper_file` + `output_file` → 简化方框
- 老节点没有 `parameters` 字段但有 `input_ports` → 简化方框
- 老节点有 `parameters` → 详细面板

全部能正常渲染。

### 5.2 canvas_layout.json

- layout 文件中不存储 `style_key` 的条目
- 加载时忽略 `style_key`，由 `auto_detect` 重新判定
- 向后兼容：layout.json 中的 style 仍保留，只是重新
- 不存 style_key 变更时以 config 为准

### 5.3 运行时动态更新

config.json 被外部修改后：
1. polling_manager.config_file_changed 信号触发
2. node_item._on_external_config_change 响应
3. 重新 auto_detect + 重新 build_anchors

## 6. 测试用例

### 6.1 单元测试

| 测试项 | 预期结果 |
|--------|---------|
| `StyleRegistry.auto_detect({"parameters": []` | detailed |
| `StyleRegistry.auto_detect({"parameters": [{"name": "x"}]` | detailed |
| `StyleRegistry.auto_detect({"input_ports": [{"source":"node"}, {"source":"node"}]` | detailed |
| `StyleRegistry.auto_detect({"input_ports": [{"source":"node"}]` | simple |
| `detect_anchors({"listen_upper_file": "x", "output_file": "x")` | 左右都有锚点 |
| `detect_anchors({"listen_upper_file": "", "output_file": "x")` | 只有输出锚点 |
| `detect_anchors({"listen_upper_file": "", "output_file": "x")` | 无锚点 |

### 6.2 集成测试

1. 创建 新建一个只有 node → 简化方框
2. 手动编辑 config.json 新增 parameters → 自动切换到详细面板
3. 手动清空 parameters → 自动回到简化方框
4. 编辑 listen_upper_file → 输入锚点消失/出现
