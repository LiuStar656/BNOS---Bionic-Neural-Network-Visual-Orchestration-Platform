## 1. 概述

菜单功能统一化系统提供了一个统一的方式来管理和创建菜单功能。核心优势：

- ✅ **单一来源**：功能定义只写一次，自动应用到所有菜单
- ✅ **i18n 支持**：内置国际化支持，使用 k 值规范
- ✅ **状态管理**：支持动态启用/禁用状态
- ✅ **上下文传递**：支持传递执行上下文（节点名、节点列表等）

## 2. 核心概念

### 2.1 ActionDefinition（功能定义）

定义一个可执行的功能单元：

| 属性 | 类型 | 说明 |
|-----|------|------|
| `id` | str | 唯一标识符（如 "node.start"） |
| `name_i18n` | str | 显示名称的 i18n key（如 "k_node_start"） |
| `category` | ActionCategory | 功能分类（NODE/CANVAS/PROJECT/VIEW/SETTINGS/TOOLS） |
| `description_i18n` | str | 描述的 i18n key |
| `shortcut_id` | str | 快捷键 ID |
| `execute_fn` | callable | 执行函数，接收 ActionContext 参数 |
| `is_enabled_fn` | callable | 启用状态检查函数 |
| `is_checked_fn` | callable | 选中状态检查函数 |

### 2.2 ActionContext（执行上下文）

传递给功能执行函数的上下文信息：

| 属性 | 类型 | 说明 |
|-----|------|------|
| `node_name` | str | 单个节点名称 |
| `node_list` | list | 节点名称列表（批量操作） |
| `group_name` | str | 组名称 |
| `extra` | dict | 额外数据 |

### 2.3 ActionRegistry（功能注册表）

单例注册表，集中管理所有功能定义。

### 2.4 ActionFactory（Action 创建工厂）

从注册表创建 QAction 的工厂类。

## 3. 快速开始

### 3.1 基本流程

```python
# 1. 导入模块
from ui.core.actions import ActionRegistry, ActionFactory, ActionDefinition, ActionContext, ActionCategory

# 2. 注册功能
ActionRegistry.register(ActionDefinition(
    id="my.custom.action",
    name_i18n="k_my_action",
    category=ActionCategory.NODE,
    execute_fn=lambda ctx: print(f"执行: {ctx.node_name}"),
))

# 3. 创建菜单 Action
menu = QMenu()
ActionFactory.create_action(main_window, "my.custom.action", menu=menu)
```

### 3.2 在菜单中使用

```python
from ui.core.actions import ActionFactory, ActionContext

# 注册（通常在初始化时调用）
register_my_actions(main_window)

# 创建菜单
menu = QMenu(self)

# 方式1：无上下文
ActionFactory.create_action(self, "node.start", menu=menu)

# 方式2：带节点上下文
ctx = ActionContext(node_name="my_node")
ActionFactory.create_action(self, "node.stop", ctx, menu=menu)

# 方式3：批量操作上下文
ctx = ActionContext(node_list=["node1", "node2", "node3"])
ActionFactory.create_action(self, "node.start", ctx, menu=menu)
```

## 4. 注册新功能

### 4.1 简单示例

```python
from ui.core.actions import ActionDefinition, ActionCategory, ActionRegistry

def register_my_actions(main_window):
    """注册自定义功能"""
    
    # 定义执行函数
    def execute_my_action(ctx):
        if ctx.node_name:
            print(f"操作节点: {ctx.node_name}")
        return True
    
    # 定义启用状态函数（可选）
    def is_my_action_enabled(ctx):
        if ctx.node_name:
            status = main_window.nodes_data.get(ctx.node_name, {}).get('status')
            return status == 'stopped'
        return True
    
    # 注册功能
    ActionRegistry.register(ActionDefinition(
        id="my.action",
        name_i18n="k_my_action",
        category=ActionCategory.NODE,
        description_i18n="_k_my_action_desc",
        shortcut_id="my_action_shortcut",  # 可选
        execute_fn=execute_my_action,
        is_enabled_fn=is_my_action_enabled,  # 可选
        requires_node=True,  # 可选：标记需要节点上下文
    ))
```

### 4.2 完整示例（带状态检查）

```python
def register_batch_actions(main_window):
    """注册批量操作功能"""
    
    def execute_batch_start(ctx):
        if ctx.node_list:
            for node_name in ctx.node_list:
                main_window.start_selected_node_by_name(node_name)
            main_window.show_toast(f"已启动 {len(ctx.node_list)} 个节点", "success")
        return True
    
    def is_batch_start_enabled(ctx):
        if ctx.node_list:
            return any(
                main_window.nodes_data.get(n, {}).get('status') == 'stopped'
                for n in ctx.node_list
            )
        return True
    
    ActionRegistry.register(ActionDefinition(
        id="node.batch_start",
        name_i18n="k_batch_start",
        category=ActionCategory.NODE,
        description_i18n="_k_batch_start_desc",
        execute_fn=execute_batch_start,
        is_enabled_fn=is_batch_start_enabled,
    ))
```

## 5. 创建菜单

### 5.1 在主菜单中使用

```python
from ui.core.actions import ActionFactory, ActionRegistry
from ui.core.actions.builtin_node_actions import register_node_actions

class MyMenuManager:
    @staticmethod
    def init_menu(main_window, menubar):
        # 注册功能
        register_node_actions(main_window)
        
        # 创建子菜单
        node_menu = ActionFactory.create_submenu(main_window, "k_menu_node", menubar=menubar)
        
        # 添加功能
        ActionFactory.create_action(main_window, "node.start", menu=node_menu)
        ActionFactory.create_action(main_window, "node.stop", menu=node_menu)
        
        # 添加分隔线
        node_menu.addSeparator()
        
        ActionFactory.create_action(main_window, "node.config", menu=node_menu)
```

### 5.2 在右键菜单中使用

```python
from ui.core.actions import ActionFactory, ActionContext
from ui.core.actions.builtin_node_actions import register_node_actions

def show_context_menu(self, node_name):
    menu = QMenu(self)
    
    # 注册功能
    register_node_actions(self.parent_window)
    
    # 创建上下文
    ctx = ActionContext(node_name=node_name)
    
    # 根据状态决定显示哪个操作
    status = self.parent_window.nodes_data.get(node_name, {}).get('status')
    if status in ('running', 'idle'):
        ActionFactory.create_action(self, "node.stop", ctx, menu)
    else:
        ActionFactory.create_action(self, "node.start", ctx, menu)
    
    menu.exec(self.mapToGlobal(event.pos()))
```

### 5.3 添加禁用标签

```python
menu = QMenu(self)

# 添加禁用的标签项（显示信息，不可点击）
ActionFactory.add_disabled_label(menu, "k_selected_count".format(count=5))

menu.addSeparator()
ActionFactory.create_action(self, "node.start", menu=menu)
```

## 6. i18n 规范

### 6.1 核心规则

| 规则 | 说明 |
|-----|------|
| 使用 k 值 | 所有显示文本必须使用 `k_` 前缀的 key |
| 延迟翻译 | ActionDefinition 中只存 key，在 ActionFactory 中翻译 |
| 双重维护 | 必须同时维护 cn/en 两个语言文件 |

### 6.2 正确示例

```python
# ✅ 正确：使用 k 值
ActionRegistry.register(ActionDefinition(
    id="node.start",
    name_i18n="k_node_start",  # ✅ k 值
    description_i18n="_k_start_node_tip",  # ✅ k 值
))

# ❌ 错误：硬编码中文
ActionRegistry.register(ActionDefinition(
    id="node.start",
    name_i18n="启动节点",  # ❌ 不要硬编码
))
```

### 6.3 添加新的 k 值

1. 在 `ui/core/strings_cn.json` 中添加中文翻译
2. 在 `ui/core/strings_en.json` 中添加英文翻译

```json
// strings_cn.json
{
  "k_my_action": "我的操作",
  "_k_my_action_desc": "执行我的自定义操作"
}

// strings_en.json
{
  "k_my_action": "My Action",
  "_k_my_action_desc": "Execute my custom action"
}
```

## 7. 最佳实践

### 7.1 功能命名规范

| 类型 | 格式 | 示例 |
|-----|------|------|
| 节点操作 | `node.xxx` | `node.start`, `node.stop`, `node.config` |
| 画布操作 | `canvas.xxx` | `canvas.clear_connections` |
| 项目操作 | `project.xxx` | `project.new`, `project.open` |
| 视图操作 | `view.xxx` | `view.node_monitor`, `view.settings` |

### 7.2 注册时机

```python
# 推荐：在应用启动时集中注册
def init_app(main_window):
    # 注册所有内置功能
    from ui.core.actions.builtin_project_actions import register_project_actions
    from ui.core.actions.builtin_node_actions import register_node_actions
    
    register_project_actions(main_window)
    register_node_actions(main_window)
    
    # 初始化菜单
    from ui.menu.menu_manager import MenuManager
    MenuManager.init_menu(main_window)
```

### 7.3 错误处理

```python
def execute_safe_action(ctx):
    try:
        # 执行操作
        result = do_something(ctx.node_name)
        
        if result:
            main_window.show_toast("操作成功", "success")
            return True
        else:
            main_window.show_toast("操作失败", "error")
            return False
            
    except Exception as e:
        main_window.show_toast(f"操作异常: {str(e)}", "error")
        logger.error(f"Action failed: {e}")
        return False
```

### 7.4 性能优化

| 优化点 | 说明 |
|-----|------|
| 延迟注册 | 只在需要时注册功能 |
| 缓存 Action | 避免重复创建相同的 QAction |
| 懒加载 | 对于不常用的功能，按需加载 |

## 8. 常见问题

### Q1: Action 创建失败怎么办？

**检查清单**：
1. ✅ 是否已调用 `register_xxx_actions(main_window)`
2. ✅ Action ID 是否正确
3. ✅ 是否在正确的上下文中调用

```python
# 调试：检查注册状态
print(ActionRegistry.get_action_ids())  # 查看所有注册的 ID
print(ActionRegistry.get("node.start"))  # 查看特定 Action
```

### Q2: 如何支持语言切换？

语言切换时需要重建菜单：

```python
def on_language_changed(self, lang):
    set_lang(lang)
    # 重新初始化菜单
    MenuManager.init_menu(self, self.menuBar())
```

### Q3: 如何传递额外参数？

使用 `extra` 字段：

```python
ctx = ActionContext(
    node_name="my_node",
    extra={"custom_data": "value", "option": True}
)

def execute_with_extra(ctx):
    custom_data = ctx.extra.get("custom_data")
    option = ctx.extra.get("option", False)
    # 使用参数...
```

### Q4: 权限控制如何实现？

使用 `requires_permission` 和 `_check_permission`：

```python
ActionRegistry.register(ActionDefinition(
    id="admin.action",
    name_i18n="k_admin_action",
    requires_permission="admin",
    execute_fn=execute_admin_action,
))

# 在 ActionRegistry 中实现权限检查
@classmethod
def _check_permission(cls, permission, context):
    # 实现权限检查逻辑
    return current_user.has_permission(permission)
```
