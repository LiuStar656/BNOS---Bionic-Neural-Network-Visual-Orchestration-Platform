# 节点进程僵尸与内存泄漏修复方案

## 一、问题总览

| 类别 | 问题 | 严重度 | 表现 |
|------|------|--------|------|
| 僵尸进程 | `_kill_process_tree` 未用 `/T`，存在竞态窗口 | P0 | 停止节点后 `main.py` 子进程变孤儿 |
| 僵尸进程 | `subprocess.PIPE` 不读取导致进程 I/O 阻塞 | P0 | 带日志输出的节点 `taskkill` 杀不动 |
| 僵尸进程 | `stop_node_process` 杀失败也改状态/删 PID | P0 | GUI 显示 stopped，进程实际活着 |
| 僵尸进程 | `check_running_processes` 跳过 `stopped` 节点 | P0 | 健康检查永不发现僵尸，状态自洽欺骗 |
| 僵尸进程 | `stop_selected_node_by_name` 先改状态后杀进程 | P0 | 用户看到停止的同时进程还在跑 |
| 内存泄漏 | `_monitor()` QThread 每次启停泄漏一个线程 | P1 | 长期运行线程数持续增长 |
| 内存泄漏 | `_node_start_workers` 异常退出无清理 | P1 | Worker 线程残留 |
| 内存泄漏 | NodeItem 信号连接从不断开 | P1 | 画布节点删除后子对象不释放 |
| 内存泄漏 | PollingManager watcher 字典无限增长 | P2 | 项目切换/节点删除不清理 |
| 内存泄漏 | `_status_callbacks` 只增不减 | P2 | 回调链越来越长 |

## 二、僵尸进程根因分析

### 2.1 完整链路（从触发到自洽欺骗）

```
用户点击「停止节点」
    │
    ▼
stop_selected_node_by_name()                    [ui/main_window/node.py:132-146]
    │
    ├─ ① UI 先行宣告死亡
    │   update_node_status(node_name, 'stopped')    ← 还没杀就改状态！
    │
    ├─ ② 10ms 后触发异步杀进程
    │   QTimer.singleShot(10, _stop_node_async)
    │
    ▼
stop_node_process()                            [ui/core/node_process.py:509-548]
    │
    ├─ ③ 尝试杀进程树（可能失败）
    │   _kill_process_tree(pid)
    │   │ 逐个 taskkill /F /PID（未用 /T）
    │   │ 从"查询"到"杀最后一批"存在数秒竞态窗口
    │   │ main.py 可在期间 fork 新子进程 → 孤儿
    │   │
    │   └─ killed = True/False
    │
    ├─ ④ 兜底扫描清理（同样可能失败）
    │   _kill_all_node_processes(node_path)
    │
    ├─ ⑤ 无论成败，强制改状态 + 删 PID
    │   node_info['status'] = 'stopped'           ← 杀没杀掉都设 stopped
    │   _delete_pid(node_path)                    ← PID 文件也删了
    │   return True, None                         ← 永远返回成功
    │
    ▼
check_running_processes()                      [ui/core/node_process.py:665-668]
    │
    └─ ⑥ 看见 status = 'stopped' → 跳过
       forever 不知道进程还活着
       
       → 日志仍在更新，GUI 永不修正 ✓ 自洽欺骗闭环
```

### 2.2 各环节详细分析

#### 环节 ①：UI 先行宣告死亡

**文件**：[`ui/main_window/node.py:137-139`](file:///f:/Bionic Neural Network Program Operating System/ui/main_window/node.py#L137-L139)


用户看到的 stop 是即时生效的，但实际杀进程发生在异步回调中。如果回调失败，UI 永远不会自行纠正。

#### 环节 ③：`_kill_process_tree` 未用 `/T`

**文件**：[`ui/core/node_process.py:300-325`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L300-L325)

```python
# 当前写法：先查进程树列表，再逐个杀
all_pids = _get_process_tree(root_pid)   # PowerShell 查询 (耗时)
for pid in all_pids:
    subprocess.run(['taskkill', '/F', '/PID', str(pid)], ...)
```

对比 `_kill_all_node_processes` 的写法：

```python
# 正确的写法，有 /T
subprocess.run(['taskkill', '/F', '/T', '/PID', str(pid)], ...)
```

`_kill_process_tree` 的竞态窗口：
- `_get_process_tree()` 用 PowerShell 查询进程树 → 耗时 1~10 秒
- 查询到"杀最后一批 PID"之间，`main.py` 可能已 fork 出子进程
- 这些子进程不在列表中 → 变成孤儿
- Windows `/T` 标志是内核级原子操作，不存在此窗口

同时 `_get_process_tree()` 函数本身约 100 行 PowerShell 字符串拼接代码，逻辑冗余。

#### 环节 ④⑤：杀失败不报

**文件**：[`ui/core/node_process.py:547-551`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L547-L551)

```python

node_info['process'] = None
node_info['status'] = 'stopped'
_delete_pid(node_path)
return True, None           # 永远返回 True
```

没有二次确认（wait + 再扫），没有错误上报，没有回滚状态。

#### 环节 ⑥：健康检查跳过 `stopped`（关键的掩埋逻辑）

**文件**：[`ui/core/node_process.py:665-668`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L665-L668)


这是整个欺骗链的最后一块拼图。即使前三步有 BUG 导致进程残留，健康检查也不会把它翻出来——因为状态已经是 `stopped`，它直接 `continue` 了。

#### 附加：`subprocess.PIPE` 阻塞

**文件**：[`ui/core/node_process.py:465-476`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L465-L476)

```python
process = subprocess.Popen(
    [python_exe, listener_py],
    ...
    stdout=subprocess.PIPE,      # 管道存在但从不读取
    stderr=subprocess.PIPE,
    text=True
)
```

当 `listener.py` 或 `main.py` 有 print 输出时，管道缓冲区满（通常 4KB~64KB），进程被操作系统挂起在 I/O 阻塞态。此时 `taskkill` 发送终止信号，进程因阻塞而无法响应。

## 三、内存泄漏分析

### 3.1 `_monitor()` QThread 泄漏

**文件**：[`ui/core/node_control_service.py:144-153`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_control_service.py#L144-L153)

```python
def _monitor(self, name, process):
    def run():
        try:
            process.wait()       # 阻塞直到进程退出
            ...
        except Exception:
            pass

    monitor_thread = QThread()
    monitor_worker = QObject()
    monitor_worker.moveToThread(monitor_thread)
    monitor_thread.started.connect(run)
    monitor_thread.finished.connect(monitor_thread.deleteLater)
    monitor_thread.start()
```

问题：
- 每次 `start_node` 创建一个 QThread，但没有保存引用 → 无法提前终止
- 如果进程被外部 kill（而非正常退出），`process.wait()` 可能永远不返回 → 线程僵尸
- 窗口关闭时，没有遍历并等待这些 monitor 线程
- QObject `monitor_worker` 没有被 `deleteLater`，线程退出后残留

### 3.2 `_node_start_workers` 无异常清理

**文件**：[`ui/main_window/node.py:85-115`](file:///f:/Bionic Neural Network Program Operating System/ui/main_window/node.py#L85-L115)

Worker 存放在 `_node_start_workers` 列表，只在 `finished` 信号回调中 `remove`。如果：
- Worker 被异常终止
- `finished` 信号因 Qt 事件循环问题未触发
- 快速关闭窗口

则列表中的对象永远不会被移除。

### 3.3 NodeItem 子对象 + 信号连接不释放

**文件**：[`ui/canvas/items/node_item.py:25-120`](file:///f:/Bionic Neural Network Program Operating System/ui/canvas/items/node_item.py#L25-L120)

每个 `NodeItem` 创建：
- 4 个子 QGraphicsTextItem（名称、语言、IN、OUT 标签）
- 1 个 QGraphicsEllipseItem（状态灯）
- 1 个 QGraphicsRectItem（展开按钮）
- 1 个 `AnchorManager` + 多个 `AnchorItem`
- 1 个 `NodeStatusWidget`（含 `QTimer`）
- 多个 `_proxy_widgets`（面板模式下的参数控件）
- `_connect_resource_monitor_signals()` 连接外部信号但从不 `disconnect`

当节点从画布场景移除时，如果 Qt 父子析构链未完全触发（如场景已在删除过程中），所有子对象泄漏。

### 3.4 PollingManager watcher 只增不删

**文件**：[`ui/core/polling_manager.py:100-105`](file:///f:/Bionic Neural Network Program Operating System/ui/core/polling_manager.py#L100-L105)


## 四、修复方案

### 4.1 P0 修复：进程终止链路

#### 修改 1：`_kill_process_tree` 改用 `/F /T`（原子操作）

**文件**：[`ui/core/node_process.py`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py)

删掉 `_get_process_tree()`（约 100 行），替换为：

```python
def _kill_process_tree(root_pid):
    """使用 /F /T 原子杀整棵进程树（内核级，无竞态窗口）"""
    try:
        if os.name == 'nt':
            result = subprocess.run(
                ['taskkill', '/F', '/T', '/PID', str(root_pid)],
                capture_output=True, timeout=10,
                creationflags=subprocess.CREATE_NO_WINDOW
            )
            return result.returncode == 0
        else:
            os.killpg(os.getpgid(root_pid), signal.SIGKILL)
            return True
    except Exception as e:
        logger.warning("终止进程树 PID=%d 失败: %s", root_pid, e)
        return False
```

`/T` 是 Windows 内核级操作，系统递归终止所有后代进程，不存在用户态查询→杀灭的竞态窗口。

#### 修改 2：stdout/stderr 改用 `DEVNULL`

**文件**：[`ui/core/node_process.py:465-476`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L465-L476)

```python
process = subprocess.Popen(
    [python_exe, listener_py],
    cwd=node_path,
    creationflags=subprocess.CREATE_NEW_PROCESS_GROUP | subprocess.CREATE_NO_WINDOW,
    stdout=subprocess.DEVNULL,
    stderr=subprocess.DEVNULL,
)
```

免除管道缓冲区满导致进程 I/O 阻塞的风险。日志已由节点自身的 `listener.log` 记录，无需 PIPE 捕获。

#### 修改 3：`stop_node_process` 确认死亡后再改状态

**文件**：[`ui/core/node_process.py:509-548`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L509-L548)

```python
def stop_node_process(node_info, force=False):
    node_path = node_info['path']
    node_name = os.path.basename(node_path)

    process = node_info.get('process')
    pid = process.pid if process else _read_pid(node_path)

    # ── 第一轮：杀进程树 ──
    killed = False
    if pid is not None:
        killed = _kill_process_tree(pid)
        if killed:
            logger.info("进程树已终止 PID=%d (%s)", pid, node_name)
        else:
            logger.warning("进程树终止失败 PID=%d (%s)，尝试兜底扫描", pid, node_name)

    # ── 兜底：进程扫描清理 ──
    if not killed:
        _kill_all_node_processes(node_path)

    # ── 二次确认：等待后重新扫描 ──
    import time
    time.sleep(0.5)
    remaining = _find_node_processes(node_path)
    confirmed_dead = len(remaining) == 0

    if confirmed_dead:
        if process:
            try:
                process.wait(timeout=2)
            except Exception:
                pass
            finally:
                try:
                    process.stdout.close()
                except Exception:
                    pass
                try:
                    process.stderr.close()
                except Exception:
                    pass
        node_info['process'] = None
        node_info['status'] = 'stopped'
        _delete_pid(node_path)
        logger.info("节点 %s 已确认停止", node_name)
    else:
        logger.error("节点 %s 终止失败，仍有 %d 个残留进程 (PIDs: %s)",
                     node_name, len(remaining), remaining)
        # 进程未死透，保持原有状态不撒谎

    return confirmed_dead, None
```

关键改进：
1. 先杀，后判定，再改状态
2. 二次确认（0.5s 等待 + 重新扫描）
3. 确认死亡后强制 close stdout/stderr
4. 失败时不改状态，不删 PID

#### 修改 5：`check_running_processes` 去掉状态过滤，引入全量扫描

**文件**：[`ui/core/node_process.py:655-710`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_process.py#L655-L710)

```python
def check_running_processes(nodes_data):
    """全量扫描所有节点进程状态，不按状态过滤
    
    核心改进：不再跳过 status='stopped' 的节点。
    这确保即使停进程失败导致的僵尸也能被健康检查发现。
    """
    dead_nodes = []
    for name, info in nodes_data.items():
        current_status = info.get('status', 'stopped')
        node_path = info['path']

        # 扫描实际运行的进程
        actual_pids = _find_node_processes(node_path)

        if actual_pids:
            # 进程确实在运行
            actual_pid = actual_pids[0]

            # 更新 PID 文件
            recorded_pid = _read_pid(node_path)
            if recorded_pid != actual_pid:
                _write_pid(node_path, actual_pid)

            # 判断运行状态
            new_status = 'running' if _listener_has_active_child(actual_pid) else 'idle'

            if current_status == 'stopped':
                # 发现僵尸进程：状态是 stopped 但进程实际活着
                logger.warning("检测到僵尸进程: %s (PIDs: %s)，状态从 stopped 修正为 %s",
                               name, actual_pids, new_status)
                info['status'] = new_status
                dead_nodes.append((name, None, new_status))
            elif current_status != new_status:
                info['status'] = new_status
                dead_nodes.append((name, None, new_status))

            # 确保内存中的 process 对象匹配
            if info.get('process') is None or info['process'].pid != actual_pid:
                info['process'] = None  # 跨会话的对象不可用
        else:
            # 没有发现运行中的进程
            if current_status in ('running', 'idle'):
                info['process'] = None
                info['status'] = 'stopped'
                _delete_pid(node_path)
                dead_nodes.append((name, 0, 'stopped'))
                logger.info("节点 %s 进程已退出", name)

    return dead_nodes
```

同时需要在代码中增加 `stopping` 状态的映射，使其在检查中也被视为应被停止的状态。

#### 修改 6：`_monitor()` 改为可追踪、可终止

**文件**：[`ui/core/node_control_service.py`](file:///f:/Bionic Neural Network Program Operating System/ui/core/node_control_service.py)

```python
class NodeControlService:
    def __init__(self):
        ...
        self._monitor_threads: Dict[str, QThread] = {}   # 新增：追踪监控线程

    def _monitor(self, name, process):
        # 先清理旧的监控线程
        if name in self._monitor_threads:
            old_thread = self._monitor_threads[name]
            if old_thread.isRunning():
                old_thread.quit()
                old_thread.wait(3000)
            del self._monitor_threads[name]

        def run():
            try:
                process.wait()
                if name in self.nodes and name not in self._active_processes:
                    self.nodes[name].status = NodeStatus.ERROR
                    self._notify(name, NodeStatus.ERROR)
            except Exception as e:
                logger.warning("节点 %s 监控异常: %s", name, e)

        monitor_thread = QThread()
        monitor_worker = QObject()
        monitor_worker.moveToThread(monitor_thread)
        monitor_thread.started.connect(run)
        monitor_thread.finished.connect(monitor_worker.deleteLater)
        monitor_thread.finished.connect(monitor_thread.deleteLater)
        monitor_thread.finished.connect(lambda: self._monitor_threads.pop(name, None))
        monitor_thread.start()

        self._monitor_threads[name] = monitor_thread

    def stop_node(self, name):
        ...
        # 在杀进程后，关闭监控线程
        if name in self._monitor_threads:
            monitor_thread = self._monitor_threads[name]
            if monitor_thread.isRunning():
                monitor_thread.quit()
                monitor_thread.wait(3000)
            del self._monitor_threads[name]
        ...

    def stop_all(self):
        for name in list(self._active_processes.keys()):
            self.stop_node(name)
        # 清理所有监控线程
        for name, thread in list(self._monitor_threads.items()):
            if thread.isRunning():
                thread.quit()
                thread.wait(3000)
        self._monitor_threads.clear()
```

```python
            if k[0] != node_path
        }
        self._config_watchers.pop(node_path, None)
        self._output_watchers.pop(node_path, None)

    def cleanup_all_watchers(self):
        """清理所有节点级监控器（项目关闭时调用）"""
        self._log_watchers.clear()
        self._config_watchers.clear()
        self._output_watchers.clear()
```
