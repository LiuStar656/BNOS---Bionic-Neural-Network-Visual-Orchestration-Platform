# 线程泄漏防护三层架构设计方案

## 1. 问题分析

### 1.1 当前架构的线程泄漏点

| 类别 | 组件 | 泄漏原因 |
|------|------|----------|
| 裸线程 | `NodeControlService._monitor_threads` | 每次监控创建一个匿名 `QThread()`（无父对象），面板关闭时不停止 |
| 裸线程 | `EdgeItem._long_press_timer` | 裸 `QTimer()` 无父对象，连线删除时泄漏 |
| 面板定时器 | `PerformancePanel._update_timer` (1000ms) | 依赖 `destroyed` 信号，`deleteLater()` 延迟时未触发 |
| 面板定时器 | `ResourceMonitorDock._update_timer` (3000ms) | 无 `closeEvent`、无 `destroyed.connect`，仅靠全局清理 |
| 面板定时器 | `NodeMonitorDock._resource_timer` (1000ms) | `closeEvent` 依赖不可靠（非顶层窗口不触发） |
| 全局单例 | `PollingManager._worker_thread` | 模块销毁顺序不确定，可能 QApplication 已被析构 |
| 全局单例 | `startup_queue._running_workers` | 退出时未显式停止 |

### 1.2 卡顿根因

7 个 Dock 面板共有 **~10 个独立 QTimer** 在主线程事件循环中周期触发：
- 每帧 3~5 个定时器回调同时抢 CPU
- 每个回调含 UI 更新（图表重绘、列表刷新、文本更新）
- 5 个面板同时打开 = 事件循环 ~15-25% 时间用于定时器回调

## 2. 设计方案：对标 Photoshop 的三层防护架构

Photoshop 核心哲学：**不创建临时线程，不做临时清理。一切资源在架构层面被管理，而不是在关闭时补救。**

```
┌─────────────────────────────────────────────────────────────┐
│  第 0 层：全局基础设施（初始化期创建，退出期销毁）              │
│  ┌─────────────────┐  ┌──────────────────┐                   │
│  │   ThreadPool     │  │ UpdateScheduler  │                   │
│  │  (固定线程池)     │  │  (单定时器调度)   │                   │
│  │  CPU核数个线程    │  │  1个QTimer·复用   │                   │
│  └────────┬────────┘  └────────┬─────────┘                   │
│           │                    │                              │
│    全局唯一实例          全局唯一实例                          │
│    启动时创建             启动时创建                           │
│    退出时销毁             退出时销毁                           │
├─────────────────────────────────────────────────────────────┤
│  第 1 层：基类契约（创建即追踪，销毁即释放）                     │
│  ┌──────────────────────────────────────┐                    │
│  │  LifecycleManaged (Mixin)            │                    │
│  │  ───────────────────────────────     │                    │
│  │  _resources: List[QObject]          │ ← 注册的所有子资源   │
│  │  _register_resource(res)            │ ← 创建时自动追踪    │
│  │  dispose() → for r in _resources:   │ ← 销毁时全部清理    │
│  │      stop() / deleteLater()         │                    │
│  └──────────────────────────────────────┘                    │
│  ┌──────────────────────────────────────┐                    │
│  │  DockPanelBase (QWidget + LCM)       │                    │
│  │  ───────────────────────────────     │                    │
│  │  自动 destroyed.connect(dispose)     │ ← 无遗漏保证       │
│  │  自动 update_scheduler.subscribe()   │ ← 统一定时器       │
│  └──────────────────────────────────────┘                    │
├─────────────────────────────────────────────────────────────┤
│  第 2 层：运行时检测（防御性，断后路）                          │
│  ┌──────────────────────────────────────┐                    │
│  │  退出时断言所有资源已释放              │                    │
│  │  if _resources: log WARNING          │                    │
│  │  开发模式: assert not _resources     │                    │
│  └──────────────────────────────────────┘                    │
└─────────────────────────────────────────────────────────────┘
```

## 3. 组件设计

### 3.1 ThreadPool — 全局固定线程池

**文件**: `ui/core/thread_pool.py`

```
ThreadPool (QObject, 单例)
├── _pool: QThreadPool (Qt 原生，固定 CPU 核数个线程)
├── run_task(fn, on_done=None) → 提交到线程池
├── cancel(task_id) → 取消任务
└── shutdown() → 等待所有任务完成
```

**设计原则**:
- 线程数 = `QThread.idealThreadCount()`（CPU 核数），不随负载变化
- 不创建也不销毁线程 —— 启动时创建，退出时销毁
- 所有需要后台执行的短任务（节点监控、进程等待）都提交到线程池
- 替代所有 `QThread()` 裸创建的代码

### 3.2 UpdateScheduler — 统一更新调度器

**文件**: `ui/core/update_scheduler.py`

```
UpdateScheduler (QObject, 单例)
├── _timer: QTimer (1000ms 固定间隔)
├── _subscribers: Dict[int, List[Callback]]
│     interval_ms → 回调列表
├── subscribe(owner, interval_ms, callback) → 注册
├── unsubscribe(owner) → 注销（owner 销毁时调用）
├── unsubscribe_all(interval_ms=None) → 全量注销
└── run() → Timer 触发 → 遍历 subscribers → 调用到期回调
```

**设计原则**:
- 只用一个 QTimer，所有面板共享
- 面板创建时 `subscribe(self, 1000, self._update_ui)`
- 面板销毁时自动 `unsubscribe(self)`（通过 `DockPanelBase.destroyed` 信号）
- 同一间隔组的回调共享触发频率，按 (当前 tick % interval) 判断是否到期
- 替代所有面板独立的 QTimer

### 3.3 LifecycleManaged — 生命周期管理 Mixin

**文件**: `ui/core/lifecycle_managed.py`

```
LifecycleManaged (Mixin)
├── _resources: List[Disposable]
├── _register_resource(resource) → 添加追踪
├── _unregister_resource(resource) → 移除追踪
├── _schedule_update(interval_ms, callback) → 注册到 UpdateScheduler
├── _run_in_thread(fn, on_done=None) → 提交到 ThreadPool
├── dispose() → 停止所有资源，注销调度器
│       ├── for r in _resources: r.stop() / r.quit()
│       ├── update_scheduler.unsubscribe(self)
│       └── _resources.clear()
└── is_disposed() → 检查是否已清理
```

**设计原则**:
- 任何创建 `QTimer` 或 `QThread` 的组件都继承此 Mixin
- `_register_resource()` 在创建资源时立即调用
- `dispose()` 在 `destroyed` 信号触发时自动调用
- `dispose()` 幂等 —— 多次调用安全
- 开发模式下 `__del__` 中 `assert not _resources`

### 3.4 DockPanelBase — Dock 面板统一基类

**文件**: `ui/core/dock_panel_base.py`

```
DockPanelBase (QWidget, LifecycleManaged)
├── 构造函数: 自动连接 destroyed.connect(self.dispose)
├── set_title(title) → 设置面板标题
├── get_title() → 获取面板标题
└── dispose() → LifecycleManaged.dispose() + 子类清理
```

**设计原则**:
- 所有 Dock 面板继承此类而非 `QWidget`
- 构造函数自动处理 `destroyed → dispose` 绑定
- 子类只需实现 `_init_ui()` 和业务逻辑
- 子类通过 `_schedule_update()` 代替 `QTimer.start()`
- 子类通过 `_run_in_thread()` 代替 `QThread()`

## 4. 改造清单

### 4.1 新建文件（4 个）

| 文件 | 职责 |
|------|------|
| `ui/core/thread_pool.py` | 全局固定线程池，替代所有裸 `QThread()` |
| `ui/core/update_scheduler.py` | 全局单定时器调度器，替代所有面板 `QTimer` |
| `ui/core/lifecycle_managed.py` | 生命周期管理 Mixin，`_register_resource()` + `dispose()` |
| `ui/core/dock_panel_base.py` | Dock 面板统一基类，组合 QWidget + LifecycleManaged |

### 4.2 改造文件（6 个）

| 文件 | 改造内容 |
|------|----------|
| `ui/panels/performance_panel.py` | 继承 `DockPanelBase`，`_update_timer` → `_schedule_update()`，`_collector_thread` → `_run_in_thread()` |
| `ui/panels/resource_monitor_dock.py` | 继承 `DockPanelBase`，`_update_timer` → `_schedule_update()` |
| `ui/panels/node_monitor_dock.py` | 继承 `DockPanelBase`，`_resource_timer` + `_list_timer` → `_schedule_update()` |
| `ui/canvas/items/edge_item.py` | `_long_press_timer` 改为有父对象，`dispose()` 中停止 |
| `ui/core/node_control_service.py` | `_monitor()` 中的裸 `QThread()` → `thread_pool.run_task()` |

## 5. 效果预期

| 指标 | 改前 | 改后 |
|------|------|------|
| 并发 QThread 数 | 不确定（每次操作创建/销毁） | 固定 CPU 核数个（启动时创建） |
| 主线程 QTimer 数 | ~10 个独立 | **1 个**统一调度 |
| 面板关闭后线程残留 | 概率性（依赖 deleteLater 时序） | **零**（dispose 同步停止） |
| 面板关闭后定时器残留 | 概率性 | **零**（dispose 同步停止） |
| 退出断言 | 无 | `dispose()` 断言所有资源已释放 |
| 事件循环定时器负载 | ~15-25%（5 面板） | ~2%（单定时器） |
| 架构一致性 | 每个面板自行管理 | 统一基类强制契约 |

| Photoshop | BNOS |
|-----------|------|

| 退出断言 `assert not _resources` | 开发模式 `__del__` 断言 |
