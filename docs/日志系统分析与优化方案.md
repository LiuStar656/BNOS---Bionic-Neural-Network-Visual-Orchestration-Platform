## 2. 发现的问题

### 2.1 🔴 严重问题：UnicodeEncodeError

**问题描述：** 日志中包含 emoji 字符（🔴📐💾），在 Windows 控制台（GBK 编码）下无法显示。

**错误示例：**
```
UnicodeEncodeError: 'gbk' codec can't encode character '\U0001f534' in position 17: illegal multibyte sequence
```

**影响：**
- 控制台输出大量错误堆栈
- 影响开发调试体验
- 可能导致日志丢失

**根本原因：**
- `StreamHandler` 使用 `sys.stdout`，在 Windows 下默认编码为 GBK
- 日志消息中包含 emoji 字符

---

### 2.2 🟡 中等问题：DEBUG 日志过多

**问题描述：** DEBUG 级别日志占比高达 71%，导致：
- 日志文件快速增长
- 难以快速定位关键信息
- 影响性能（频繁磁盘写入）

**典型 DEBUG 日志：**
```
[2026-06-12 23:51:10] DEBUG BNOS (polling_manager.py:132): Registered polling task: node_health (interval: 2s)
[2026-06-12 23:51:21] DEBUG BNOS (edge_item.py:118): [EdgeItem] _setup_anchor_binding: desired_target_port=default
```

---

### 2.3 🟡 中等问题：日志轮转配置不合理

**问题描述：**
- 单文件 5MB 在高频 DEBUG 日志下很快达到
- 只有 3 个备份，历史日志保留时间短
- 没有按日期分割日志

---

### 2.4 🟢 轻微问题：日志格式不一致

**问题描述：**
- 控制台格式：`[时间] 级别 消息`
- 文件格式：`[时间] 级别 名称 (文件:行号): 消息`
- 控制台缺少文件位置信息，调试时不便

---

### 2.5 🟢 轻微问题：缺少日志分类

**问题描述：**
- 所有日志写入同一文件
- 没有按模块（节点、画布、IPC 等）分类
- 难以单独分析某个模块的问题

---

### 2.6 🟢 轻微问题：性能影响

**问题描述：**
- 每条日志都立即写入磁盘
- 高频 DEBUG 日志可能影响主线程性能
- 没有异步日志机制

## 3. 优化方案

### 3.1 修复 UnicodeEncodeError（优先级：P0）

**方案 A：移除日志中的 emoji 字符**

```python
# 修改前
logger.info("🔴 开始恢复面板状态")

# 修改后
logger.info("[PANEL] 开始恢复面板状态")
```

**方案 B：配置控制台编码**

```python
# 在 logger.py 中添加
import sys
import io

# 强制使用 UTF-8 编码
if sys.platform == 'win32':
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')
    sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8', errors='replace')
```

**方案 C：自定义 Handler 处理编码错误**

```python
class SafeStreamHandler(logging.StreamHandler):
    """安全的流处理器，处理编码错误"""
    
    def emit(self, record):
        try:
            super().emit(record)
        except UnicodeEncodeError:
            # 替换无法编码的字符
            record.msg = record.msg.encode('gbk', errors='replace').decode('gbk')
            super().emit(record)
```

**推荐：** 方案 A + 方案 C 组合使用

### 3.2 优化日志级别控制（优先级：P1）

**方案：动态日志级别 + 配置文件控制**

```python
# app_config.json 中添加
{
    "logging": {
        "console_level": "INFO",
        "file_level": "INFO",  // 生产环境改为 INFO
        "debug_modules": ["polling_manager", "canvas_view"]  // 仅特定模块开启 DEBUG
    }
}
```

**实现：**
```python
def setup_logger(name: str = "BNOS", config: dict = None) -> logging.Logger:
    logger = logging.getLogger(name)
    
    # 从配置读取日志级别
    console_level = getattr(logging, config.get('console_level', 'INFO'))
    file_level = getattr(logging, config.get('file_level', 'DEBUG'))
    
    console_handler.setLevel(console_level)
    file_handler.setLevel(file_level)
    
    return logger
```

### 3.3 改进日志轮转策略（优先级：P1）

**方案：按日期 + 大小双重轮转**

```python
from logging.handlers import TimedRotatingFileHandler

# 按日期轮转，每天一个文件
file_handler = TimedRotatingFileHandler(
    log_dir / "bnos_console.log",
    when='midnight',      # 每天午夜轮转
    interval=1,           # 每天一个文件
    backupCount=7,        # 保留 7 天
    encoding='utf-8'
)

# 或者使用 ConcurrentLogHandler 支持多进程
# pip install ConcurrentLogHandler
from cloghandler import ConcurrentRotatingFileHandler

file_handler = ConcurrentRotatingFileHandler(
    log_dir / "bnos_console.log",
    maxBytes=10 * 1024 * 1024,  # 10MB
    backupCount=5,
    encoding='utf-8'
)
```

### 3.4 添加日志分类（优先级：P2）

**方案：多文件日志处理器**

```python
def setup_module_logger(module_name: str) -> logging.Logger:
    """为特定模块创建独立的日志器"""
    logger = logging.getLogger(f"BNOS.{module_name}")
    
    # 模块专属日志文件
    file_handler = RotatingFileHandler(
        log_dir / f"bnos_{module_name}.log",
        maxBytes=5 * 1024 * 1024,
        backupCount=3,
        encoding='utf-8'
    )
    logger.addHandler(file_handler)
    
    return logger

# 使用
canvas_logger = setup_module_logger("canvas")
node_logger = setup_module_logger("node")
ipc_logger = setup_module_logger("ipc")
```

### 3.5 异步日志写入（优先级：P2）

**方案：使用 QueueHandler 实现异步日志**

```python
from logging.handlers import QueueHandler, QueueListener
import queue

# 创建日志队列
log_queue = queue.Queue(-1)  # 无限大小

# 队列处理器（在后台线程中运行）
queue_handler = QueueHandler(log_queue)
file_handler = RotatingFileHandler(...)

listener = QueueListener(log_queue, file_handler)
listener.start()

# 主日志器使用队列处理器
logger.addHandler(queue_handler)

# 应用退出时停止监听器
import atexit
atexit.register(listener.stop)
```

### 3.6 敏感信息过滤（优先级：P2）

**方案：自定义 Filter**

```python
class SensitiveDataFilter(logging.Filter):
    """过滤敏感信息"""
    
    SENSITIVE_PATTERNS = [
        r'password["\']?\s*[:=]\s*["\']?[^"\',\s]+',
        r'token["\']?\s*[:=]\s*["\']?[^"\',\s]+',
        r'api_key["\']?\s*[:=]\s*["\']?[^"\',\s]+',
    ]
    
    def filter(self, record):
        import re
        for pattern in self.SENSITIVE_PATTERNS:
            record.msg = re.sub(pattern, '***REDACTED***', str(record.msg))
        return True

# 添加过滤器
logger.addFilter(SensitiveDataFilter())
```

## 5. 推荐的日志规范

### 5.1 日志级别使用规范

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| **DEBUG** | 详细调试信息，仅开发时使用 | `logger.debug("变量值: %s", value)` |
| **INFO** | 正常业务流程 | `logger.info("节点 %s 已启动", name)` |
| **WARNING** | 潜在问题，不影响运行 | `logger.warning("配置项缺失，使用默认值")` |
| **ERROR** | 错误，但程序可继续运行 | `logger.error("加载文件失败: %s", path)` |
| **CRITICAL** | 严重错误，程序可能崩溃 | `logger.critical("数据库连接失败")` |

### 5.2 日志消息规范

```python
# ✅ 好的日志
logger.info("[模块] 操作描述: %s", detail)
logger.error("[模块] 操作失败: %s, 原因: %s", item, reason)

# ❌ 不好的日志
logger.info("🔴 开始处理...")  # 包含 emoji
logger.debug(str(complex_object))  # 过大的对象
logger.info("处理完成")  # 缺少上下文
```

### 5.3 性能优化建议

```python
# ✅ 使用 % 格式化（惰性求值）
logger.debug("复杂计算结果: %s", expensive_function())

# ❌ 使用 f-string（立即求值）
logger.debug(f"复杂计算结果: {expensive_function()}")  # 即使不输出也会执行

# ✅ 检查日志级别后再执行复杂操作
if logger.isEnabledFor(logging.DEBUG):
    logger.debug("详细数据: %s", generate_detailed_report())
```
